## 一、引言

算法设计的核心挑战在于：在有限的计算资源下，找到足够正确且足够高效的解法。贪心策略和动态规划是解决最优化问题的两大主流范式，二者在表面上都追求"局部最优"，然而其内在逻辑截然不同。

贪心法在每一步选取当前看来最优的决策，且不回头；动态规划则通过记录所有子问题的最优解，自底向上或自顶向下地构建全局最优。正确区分两者的适用场景，既需要直觉，也需要严格的理论支撑。

本文选取的三道题目具有递进关系：LIS（300）展示了O(n²)DP与O(n log n)贪心+二分的对比；LCS（1143）展示了二维DP的状态设计；Coin Change（322）则揭示了贪心在无最优子结构时的失效。

## 二、基础理论

### 2.1 复杂度的严格定义

设 f(n) 和 g(n) 为定义在正整数上的函数。

**定义（大O上界）：** 若存在正常数 c 和 n₀，使得对所有 n ≥ n₀ 有 f(n) ≤ c·g(n)，则称 f(n) = O(g(n))。

**定义（Ω下界）：** 若存在正常数 c 和 n₀，使得对所有 n ≥ n₀ 有 f(n) ≥ c·g(n)，则称 f(n) = Ω(g(n))。

**定义（Θ紧界）：** 若 f(n) = O(g(n)) 且 f(n) = Ω(g(n))，则称 f(n) = Θ(g(n))。

**最坏复杂度**描述算法在所有可能输入中最差情况的运行时间，是算法鲁棒性的保证；**平均复杂度**在某一概率分布假设下给出期望代价；**最优复杂度（下界）**则说明对于某类问题，任何算法都无法突破的时间壁垒。

**命题：** 基于比较的排序算法最坏复杂度为 Ω(n log n)。

**证明（决策树论证）：** n 个元素的全排列有 n! 种。任何基于比较的排序可建模为一棵二叉决策树，每个叶节点对应一种排列。叶节点数至少为 n!，树高 h 满足 2^h ≥ n!，故 h ≥ log₂(n!) = Θ(n log n)（由 Stirling 近似）。因此最坏情况至少需要 Ω(n log n) 次比较。□

这一下界的意义在于：当我们后续将 LIS 问题的复杂度优化至 O(n log n) 时，某种意义上已触及了"排序级"的下界。

### 2.2 动态规划的两大核心性质

动态规划正确性依赖两个条件：

1. **最优子结构（Optimal Substructure）：** 问题的最优解可由子问题的最优解构造。
2. **重叠子问题（Overlapping Subproblems）：** 递归求解过程中，相同子问题被反复计算，记忆化或制表可避免重复。

若问题缺乏最优子结构，贪心或DP都无法直接给出正确答案。

### 2.3 贪心法的适用条件

贪心正确的充分条件通常通过**交换论证（Exchange Argument）**证明：假设存在一个不采用贪心选择的最优解，通过将其局部替换为贪心选择，可以构造出同样不劣的新最优解，从而归纳证明贪心全局最优。

## 三、问题一：最长递增子序列（LeetCode 300）

### 3.1 问题描述

给定整数数组 `nums`，找到其中最长严格递增子序列的长度。子序列不要求连续。

**示例：** `nums = [10, 9, 2, 5, 3, 7, 101, 18]`，输出 `4`，对应子序列 `[2, 3, 7, 101]`。

### 3.2 动态规划解法（O(n²)）

**状态定义：** 设 `dp[i]` 为以 `nums[i]` 结尾的最长递增子序列长度。

**状态转移：**
```
dp[i] = max(dp[j] + 1)，对所有 j < i 且 nums[j] < nums[i]
```

**初始值：** `dp[i] = 1`（每个元素自身构成长度为1的子序列）。

**答案：** `max(dp[0], dp[1], ..., dp[n-1])`。

```python
def lengthOfLIS_dp(nums):
    n = len(nums)
    dp = [1] * n
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)
```

**复杂度分析：** 双重循环，时间复杂度 Θ(n²)，空间复杂度 Θ(n)。

**正确性：** 对 i 归纳。当 i=0，dp[0]=1 显然正确。设所有 j < i 的 dp[j] 均正确，则 dp[i] 枚举了所有可以接在 nums[i] 之前的末尾元素，取最大值加1，得到以 nums[i] 结尾的最优解。由最优子结构，结论成立。

### 3.3 贪心 + 二分优化（O(n log n)）

**核心思想（耐心排序，Patience Sorting）：** 维护一个数组 `tails`，`tails[k]` 表示所有长度为 k+1 的递增子序列中，末尾元素的最小值。对每个新元素，用二分查找在 `tails` 中找到第一个大于等于它的位置并替换，若不存在则追加。

```python
import bisect

def lengthOfLIS_greedy(nums):
    tails = []
    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num
    return len(tails)
```

**为何贪心正确：** 贪心策略保证 `tails` 始终严格递增，且 `tails[k]` 是所有长度为 k+1 子序列末尾的最小可能值。替换操作不改变 `tails` 的长度（即不影响当前最优子序列长度），但使未来延伸的可能性最大化。通过交换论证可以严格证明：任何替换为非最小末尾的策略，其最终 LIS 长度不优于贪心。

**复杂度分析：** 外循环 n 次，每次二分查找 O(log n)，总时间复杂度 O(n log n)，空间 O(n)。

### 3.4 对抗策略与下界

从**对抗策略（Adversary Argument）**角度看，对手可以构造输入使任何算法都必须比较 Ω(n log n) 次才能确定 LIS 长度（若算法基于比较）。具体地，对手选择完全随机排列的输入，使得没有"捷径"可绕过有效的比较。因此 O(n log n) 的解法在比较模型下是最优的。

## 四、问题二：最长公共子序列（LeetCode 1143）

### 4.1 问题描述

给定两个字符串 `text1` 和 `text2`，返回它们最长公共子序列（LCS）的长度。

**示例：** `text1 = "abcde"`，`text2 = "ace"`，输出 `3`，对应 `"ace"`。

### 4.2 二维动态规划

**状态定义：** 设 `dp[i][j]` 为 `text1[0..i-1]` 与 `text2[0..j-1]` 的 LCS 长度。

**状态转移：**
```
若 text1[i-1] == text2[j-1]：dp[i][j] = dp[i-1][j-1] + 1
否则：                        dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

**初始值：** `dp[0][j] = dp[i][0] = 0`（空串与任意串的 LCS 为0）。

```python
def longestCommonSubsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i-1] == text2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]
```

**复杂度分析：** 时间复杂度 Θ(mn)，空间复杂度 Θ(mn)，可优化至 O(min(m,n))（滚动数组）。

### 4.3 最优子结构的验证

设 `LCS(i,j)` 为 `text1[0..i-1]` 与 `text2[0..j-1]` 的最优解。  
- 若 `text1[i-1] == text2[j-1]`：该字符必然在某个最优LCS中，`LCS(i,j) = LCS(i-1,j-1) + 1`。  
- 否则：最优LCS不同时包含两者的最后一个字符，即要么不含 `text1[i-1]`，要么不含 `text2[j-1]`，故 `LCS(i,j) = max(LCS(i-1,j), LCS(i,j-1))`。

两种情况均满足最优子结构，DP 正确。

### 4.4 为何贪心失效

若用贪心策略（如每次选两个字符串中最早公共字符），考虑：

```
text1 = "abcbdab"
text2 = "bdcaba"
```

贪心可能先选 'b'（位置0和0），再选 'd'（位置4和1），再选 'a'（位置6和4），得到长度3。  
而真正的 LCS 为 "bcba" 或 "bdab"，长度4。贪心的局部选择破坏了全局最优性，原因在于提前消耗了后续可能更优的匹配机会。

## 五、问题三：零钱兑换（LeetCode 322）

### 5.1 问题描述

给定面值数组 `coins` 和目标金额 `amount`，求凑成 `amount` 所需的最少硬币数，若无法凑成返回 -1。

**示例：** `coins = [1, 5, 11]`，`amount = 15`，最优解为 `[5, 5, 5]`，输出 `3`。

### 5.2 贪心的失败

直觉上，每次选最大面值的硬币看起来最优。对 `coins = [1, 5, 11]`，`amount = 15`：

- 贪心：选11，剩4；选1×4，共5枚。  
- 最优：选5×3，共**3枚**。

贪心失败，因为硬币面值不满足"每次贪心选择后，剩余子问题的最优解与整体最优解兼容"的条件。即**缺乏贪心选择性质**。

### 5.3 动态规划解法

**状态定义：** `dp[i]` 为凑成金额 i 所需最少硬币数。

**状态转移：**
```
dp[i] = min(dp[i - coin] + 1)，对所有 coin ∈ coins 且 i - coin ≥ 0
```

**初始值：** `dp[0] = 0`，`dp[i] = +∞`（i > 0）。

```python
def coinChange(coins, amount):
    INF = float('inf')
    dp = [INF] * (amount + 1)
    dp[0] = 0
    for i in range(1, amount + 1):
        for coin in coins:
            if i - coin >= 0 and dp[i - coin] != INF:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    return dp[amount] if dp[amount] != INF else -1
```

**复杂度分析：** 外层循环 O(amount)，内层循环 O(|coins|)，总时间 O(amount × |coins|)，空间 O(amount)。

**正确性：** 最优子结构成立——若最优方案选了面值 c，则剩余 amount-c 的最优方案即为 dp[amount-c]，否则可替换得到更优方案，矛盾。

### 5.4 平摊分析视角

在 DP 的迭代计算中，每个状态 `dp[i]` 被计算一次，但在后续状态的转移中被读取多次（最多 |coins| 次）。若从**平摊分析（Amortized Analysis）**的角度看，对每枚硬币面值 c，它对所有 `dp[c..amount]` 各贡献一次读操作，总读写操作为 Θ(amount × |coins|)，与朴素分析一致，但揭示了每个子问题的"被使用次数"均匀分布，不存在热点状态，算法在实践中具有良好的cache行为。

更典型的平摊分析场景是动态数组的扩容：每次扩容代价为 O(n)，但均摊到 n 次插入操作后，每次插入的均摊代价为 O(1)（势能法）。

## 六、算法比较与综合心得

### 6.1 三道题的对比总结

| 问题 | 贪心可行？ | DP类型 | 时间复杂度 | 关键性质 |
|---|---|---|---|---|
| LIS (300) | 可行（需二分辅助） | 一维DP | O(n log n) | 最优子结构 + 贪心选择性质 |
| LCS (1143) | 不可行 | 二维DP | O(mn) | 最优子结构，无贪心选择性质 |
| Coin Change (322) | 不可行（一般情况） | 一维DP | O(amount·|coins|) | 最优子结构，贪心反例存在 |

### 6.2 对"贪心 vs 动态规划"判断的心得

判断能否用贪心，核心在于**能否证明贪心选择性质**：即存在一个最优解，它包含贪心算法在第一步的选择。证明手段通常是交换论证。如果无法构造这个证明，而问题又具有最优子结构，则应转向动态规划。

LIS问题的有趣之处在于：它既有O(n²)的DP解法，又有O(n log n)的贪心+二分解法。两者并不矛盾——DP保证了正确性的理论基础，而贪心（维护最小末尾的策略）在DP的框架下提供了一种等价但更高效的计算方式。这说明DP和贪心并非非此即彼，有时贪心是DP的一种高效实现。

### 6.3 分治法的角色

在LIS的O(n log n)解法中，二分查找本质上是一种**分治思想**：将 `tails` 数组的搜索空间每次减半。二分搜索满足递归关系：

```
T(n) = T(n/2) + O(1)
```

由主定理（Master Theorem），a=1，b=2，f(n)=O(1)，log_b(a) = 0，故 T(n) = O(log n)。这正是每次查找的复杂度。

### 6.4 对抗策略的启示

在设计算法时，**对抗策略**不仅是证明下界的工具，更是一种思维方式：想象有一个"对手"专门构造让你的算法表现最差的输入。这种思维帮助我们识别算法的薄弱环节。例如，在LIS问题中，完全降序数组使得O(n²)的DP每次内层循环几乎无有效转移，而贪心+二分则依然稳定在O(n log n)。在设计鲁棒算法时，始终考虑最坏情况是必要的。

### 6.5 动态规划的空间优化

LCS问题中，`dp[i][j]` 只依赖 `dp[i-1][j-1]`、`dp[i-1][j]`、`dp[i][j-1]`，即只需前一行数据。利用滚动数组，可将空间从 O(mn) 优化至 O(min(m,n))：

```python
def longestCommonSubsequence_opt(text1, text2):
    if len(text1) < len(text2):
        text1, text2 = text2, text1
    m, n = len(text1), len(text2)
    prev = [0] * (n + 1)
    for i in range(1, m + 1):
        curr = [0] * (n + 1)
        for j in range(1, n + 1):
            if text1[i-1] == text2[j-1]:
                curr[j] = prev[j-1] + 1
            else:
                curr[j] = max(prev[j], curr[j-1])
        prev = curr
    return prev[n]
```

这一优化体现了**工程与理论的结合**：在不改变时间复杂度的前提下，大幅降低空间消耗，在处理长字符串时具有实际意义。

## 七、结论

通过对LIS、LCS和Coin Change三道经典问题的深入分析，本文系统阐述了以下核心观点：

1. **复杂度分析是算法评价的基石。** 严格的大O定义、下界证明（决策树论证、对抗策略）使我们对算法效率有客观认知，避免被直觉误导。

2. **动态规划的正确性依赖最优子结构。** 验证最优子结构是将直觉转化为正确DP方程的关键步骤，不可跳过。

3. **贪心是有条件的捷径。** 贪心选择性质的存在使贪心比DP更高效，但这一性质必须通过交换论证严格验证，而非凭感觉判断。

4. **分治法是提升效率的利器。** 二分查找将LIS从O(n²)优化至O(n log n)，体现了分治思想在非分治问题中的创造性应用。

5. **平摊分析揭示均匀代价分布。** 对于含有批量操作的算法，平摊视角提供了比最坏分析更紧的界，有助于理解算法的实际行为。

算法学习的真正价值，不在于记住特定问题的解法，而在于建立一套分析框架：识别问题结构、选择合适范式、严格证明正确性、精确分析复杂度。这三道题的学习历程，正是对这一框架的一次完整实践。
