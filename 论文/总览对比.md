# RLHF / PPO / DPO / GRPO 总览对比

---

## 演进脉络

```
预训练 LLM（只会续写文本）
       ↓
  [2022] RLHF + PPO（InstructGPT）
  → 解决了：如何让 LLM 听从人类指令
  → 遗留了：训练复杂、显存贵、需要奖励模型
       ↓
  [2023] DPO（Stanford）
  → 解决了：去掉 RM 和 PPO，直接用偏好数据微调
  → 遗留了：离线方法，推理能力提升有限
       ↓
  [2024] GRPO（DeepSeek）
  → 解决了：去掉 Critic，用组内竞争代替价值估计
  → 突破了：数学/代码推理能力，涌现长思维链
```

---

## 四种方法核心对比表

| 维度 | RLHF | PPO | DPO | GRPO |
|------|------|-----|-----|------|
| **角色** | 整体框架 | RLHF 的优化算法 | RLHF 的替代方案 | PPO 的改进版 |
| **需要 RM** | 是 | 是（作为环境） | 否（隐式） | 是（可自动） |
| **需要 Critic** | 是 | 是 | 否 | **否** |
| **在线/离线** | 在线 | 在线 | 离线 | 在线 |
| **模型数量** | 4 | 4 | 2 | 3 |
| **训练稳定性** | 中 | 中 | 高 | 高 |
| **推理能力提升** | 中 | 中 | 弱 | **强** |
| **代表模型** | InstructGPT | ChatGPT | Zephyr, Llama系列 | DeepSeek-R1 |

---

## 各方法的适用场景

```
有大量人工偏好标注数据
+ 计算资源充足
+ 追求通用对齐效果
→ 选 RLHF + PPO

有偏好对数据（A比B好）
+ 计算资源有限
+ 不需要在线探索
→ 选 DPO

需要提升数学/代码/推理
+ 奖励可以自动验证（对/错）
+ 希望涌现思维链
→ 选 GRPO
```

## 关键公式速查

### RLHF 奖励模型损失
$$\mathcal{L}_{RM} = -\mathbb{E} \left[ \log \sigma(r(x, y_w) - r(x, y_l)) \right]$$

### PPO Clip 损失
$$\mathcal{L}^{CLIP} = \mathbb{E}_t \left[ \min\left( r_t \hat{A}_t,\ \text{clip}(r_t, 1-\epsilon, 1+\epsilon) \hat{A}_t \right) \right]$$

### DPO 损失
$$\mathcal{L}_{DPO} = -\mathbb{E} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right]$$

### GRPO 优势估计
$$\hat{A}_i = \frac{r_i - \text{mean}(r_1,...,r_G)}{\text{std}(r_1,...,r_G)}$$

## 一句话记忆法

| 方法 | 一句话 |
|------|--------|
| **RLHF** | 用人类打分训练"懂规矩"的 AI |
| **PPO** | 每次小步更新，防止策略崩溃 |
| **DPO** | 直接告诉模型"好的像这个，差的别这样" |
| **GRPO** | 让模型自己和自己比，组内竞争激发推理 |
