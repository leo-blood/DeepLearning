# GRPO 课件：群体相对策略优化

> 论文：*DeepSeekMath: Pushing the Limits of Mathematical Reasoning in Open Language Models*（DeepSeek，2024）
> GRPO 在 DeepSeek-R1 中被大规模应用，推动了 LLM 推理能力的突破

---

## 1. 背景：PPO 的显存瓶颈

RLHF 中使用 PPO 需要同时维护 **4 个模型**：

```
1. Actor（策略模型，被训练）        ← 最大，需要梯度
2. Critic（价值模型，估计 baseline） ← 和 Actor 一样大
3. 奖励模型 RM                       ← 用于打分
4. 参考模型 π_ref（SFT 基线）        ← 用于 KL 惩罚
```

对于 70B 甚至更大的模型，4 个模型同时加载几乎不可能。

**GRPO 的解法**：去掉 Critic（价值网络），用**组内相对奖励**代替优势估计。

---

## 2. GRPO 核心思想

### 关键创新：Group-based Baseline

对同一个 prompt $x$，让模型**采样一组回答**（如 G=8 个）：

$$\{y_1, y_2, y_3, ..., y_G\} \sim \pi_{\theta_{old}}(\cdot | x)$$

每个回答都得到奖励打分：$\{r_1, r_2, r_3, ..., r_G\}$

**优势估计**：用组内均值和标准差做归一化：

$$\hat{A}_i = \frac{r_i - \text{mean}(\{r_1,...,r_G\})}{\text{std}(\{r_1,...,r_G\})}$$

- $\hat{A}_i > 0$：这个回答比组内平均水平好
- $\hat{A}_i < 0$：比组内平均水平差
- **不需要 Critic 网络**，只需要组内比较！

---

## 3. GRPO 损失函数

$$\mathcal{L}_{GRPO}(\theta) = -\frac{1}{G} \sum_{i=1}^{G} \left[ \min\left( r_i(\theta) \hat{A}_i,\ \text{clip}(r_i(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_i \right) - \beta \cdot KL[\pi_\theta \| \pi_{ref}] \right]$$

其中：
- $r_i(\theta) = \frac{\pi_\theta(y_i|x)}{\pi_{\theta_{old}}(y_i|x)}$ — 重要性采样比率（和 PPO 一样）
- $\hat{A}_i$ — 组内归一化优势（代替 PPO 的 Critic 估计）
- $\text{clip}$ — PPO 的裁剪机制（防止更新过大）
- KL 惩罚 — 不偏离参考模型太远

---

## 4. 对比图解：PPO vs GRPO

### PPO 的优势估计

```
prompt x → 生成 y → RM 打分 r
                          ↓
                    Critic(s) → 预测 V(s)
                          ↓
              A = r - V(s)  ← 需要一个独立的 Critic 模型
```

### GRPO 的优势估计

```
prompt x → 同时生成 y_1, y_2, ..., y_G（一组）
                          ↓
              RM 分别打分 r_1, r_2, ..., r_G
                          ↓
         A_i = (r_i - mean(r)) / std(r)  ← 组内相对比较，无需 Critic
```

## 5. 直观例子

假设 G=4，prompt 是"解方程 x² - 5x + 6 = 0"：

```
回答 y_1：x=2 或 x=3（正确）          r_1 = 1.0
回答 y_2：x=2 或 x=4（部分错误）       r_2 = 0.3
回答 y_3：x=-2 或 x=-3（符号错误）     r_3 = 0.1
回答 y_4：无法求解（拒绝回答）         r_4 = 0.0

mean(r) = 0.35，std(r) = 0.41

归一化优势：
  A_1 = (1.0 - 0.35) / 0.41 = +1.59  ← 大力鼓励
  A_2 = (0.3 - 0.35) / 0.41 = -0.12  ← 轻微抑制
  A_3 = (0.1 - 0.35) / 0.41 = -0.61  ← 中度抑制
  A_4 = (0.0 - 0.35) / 0.41 = -0.85  ← 强烈抑制
```

模型学到：**在这个 prompt 下，正确的推理步骤应该得到强化。**

## 6. 奖励函数设计（DeepSeek-R1 中）

GRPO 的效果很大程度依赖奖励函数的设计。DeepSeek-R1 用了两类奖励：

### 准确性奖励（Accuracy Reward）
```
数学题：答案完全正确 → r = 1，否则 r = 0
代码题：测试用例全部通过 → r = 1，否则 r = 0
```

### 格式奖励（Format Reward）
```
要求模型在 <think>...</think> 中展示推理过程
符合格式 → 小额正奖励
不符合格式 → 小额负奖励
```

> 关键：**奖励是可验证的（verifiable reward）**，不需要人工标注，可以自动计算。

---

## 7. GRPO 为什么能涌现推理能力？

GRPO 中，模型要在组内"竞争"：

1. 对同一道数学题，模型生成 8 个不同的解题路径
2. 有的路径推理正确（高分），有的路径错误（低分）
3. 通过反复优化，模型学会了**用更可靠的推理步骤**

```
初始：8 个回答，3 个正确 → 正确率 37.5%
训练后：8 个回答，7 个正确 → 正确率 87.5%
```

这就是 DeepSeek-R1 中出现"长思维链（Chain of Thought）"的原因——模型自发学会了"多想一步"。

---

## 8. 三步法与 GRPO 的关系

DeepSeek-R1 的完整训练流程：

```
Step 1: SFT（少量种子数据，建立格式）
         ↓
Step 2: GRPO（大规模强化学习，涌现推理）
         ↓
Step 3: 拒绝采样 + SFT（提炼高质量思维链）
         ↓
Step 4: GRPO（再次强化，兼顾通用能力）
```

GRPO 是核心引擎，驱动推理能力的飞跃。

## 9. 对比四种方法

| 特性 | RLHF(PPO) | DPO | GRPO |
|------|-----------|-----|------|
| 需要奖励模型 RM | 是 | 否 | 是（但可自动） |
| 需要 Critic 网络 | 是 | 否 | **否** |
| 模型数量 | 4 | 2 | **3**（Actor + RM + Ref） |
| 在线/离线 | 在线 | 离线 | **在线** |
| 适合场景 | 通用对齐 | 偏好学习 | **推理/数学/代码** |
| 奖励来源 | 人工标注 | 偏好对 | **可验证规则** |
| 显存压力 | 极高 | 中 | **中高** |

---

## 10. GRPO 的优缺点

### 优点
- **省显存**：去掉 Critic，比 PPO 少一个大模型
- **稳定**：组内归一化降低方差，训练更平稳
- **自动奖励**：数学/代码场景可用规则自动打分，无需人工
- **效果强**：DeepSeek-R1 证明了其在推理任务上的突破性效果

### 缺点
- **需要大量采样**：每个 prompt 要生成 G 个回答，计算量大
- **组大小 G 敏感**：G 太小则方差大，G 太大则慢
- **适用范围**：最适合有明确对错的任务（数学/代码），主观任务效果不确定

---

## 11. 直观类比

> GRPO 就像一场**小组竞赛**：
> - 老师出一道题（prompt）
> - 全班同时作答（G 个回答）
> - 打分后排名，得分高于平均的同学获得奖励，低于平均的被扣分
> - **不需要一个专门的"评分预测员"（Critic）**，大家互相比较就够了
> - 班级整体水平越来越高（推理能力涌现）

## 12. 关键词速查

| 术语 | 含义 |
|------|------|
| Group Sampling | 对同一 prompt 采样多个回答 |
| Group Baseline | 用组内均值作为 baseline，代替 Critic |
| Verifiable Reward | 可自动验证的奖励（如数学答案对错） |
| Chain of Thought | 思维链，推理过程的逐步展示 |
| On-policy | 在线方法，用当前策略实时采样数据 |
| Rejection Sampling | 拒绝采样，只保留高质量输出用于后续训练 |
