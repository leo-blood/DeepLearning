# RLHF 课件：用人类反馈训练更好的 AI

> 论文：*Training language models to follow instructions with human feedback*（InstructGPT，OpenAI，2022）

---

## 1. 为什么需要 RLHF？

大语言模型（LLM）预训练后只会"续写文本"，并不懂得"什么是好的回答"。

| 问题 | 举例 |
|------|------|
| 说有害内容 | "如何制作炸弹？" → 模型照样回答 |
| 答非所问 | 问"总结一下" → 模型继续延伸故事 |
| 过于冗长 | 简单问题给出几千字废话 |

**核心思想**：让人类给模型的输出打分，然后用这个分数来优化模型。

---

## 2. RLHF 三步流程

```
Step 1: 监督微调 (SFT)
         ↓
Step 2: 训练奖励模型 (RM)
         ↓
Step 3: 强化学习优化 (PPO)
```

### Step 1：监督微调（SFT）

- 收集高质量的"提示-回答"对（人工标注）
- 用这批数据直接 fine-tune 预训练模型
- 目标：让模型学会基本的指令跟随格式

```
输入：请写一首关于春天的诗
输出：（人工写好的高质量诗）
↓ 监督学习微调
SFT 模型
```

### Step 2：训练奖励模型（RM）

- 对同一个提示，让 SFT 模型生成多个不同回答（如 4 个）
- 人工标注员对这些回答**排序**（哪个最好、哪个最差）
- 用排序数据训练一个**奖励模型**，让它学会给回答打分

```
提示："解释黑洞"
回答 A：详细准确  ← 人类排第 1
回答 B：有点错误  ← 人类排第 2
回答 C：太短了    ← 人类排第 3
回答 D：废话连篇  ← 人类排第 4

↓ 对比学习训练

奖励模型 RM：能给任意回答打出分数
```

> **奖励模型损失函数**：
> $$\mathcal{L}_{RM} = -\mathbb{E}_{(x,y_w,y_l)} \left[ \log \sigma(r(x, y_w) - r(x, y_l)) \right]$$
> 其中 $y_w$ 是更好的回答，$y_l$ 是更差的回答，$r$ 是奖励分数。
> 直觉：好回答的分数要比差回答高。

### Step 3：强化学习优化（PPO）

- 把 SFT 模型当作 **Actor（策略）**
- 用 RM 的分数当作 **奖励信号**
- 用 PPO 算法优化 Actor，让它生成的回答得分更高
- 同时加入 **KL 散度惩罚**，防止模型偏离太远变成"只会骗分"

```
提示 x → SFT模型(Actor) → 回答 y
                               ↓
                         奖励模型 RM → 分数 r
                               ↓
              PPO 更新 Actor，让下次分数更高
              + KL惩罚：别跑太远（保持语言能力）
```

---

## 3. KL 惩罚：为什么重要？

如果只追求 RM 分数，模型会**过拟合奖励模型**（reward hacking）：

> 比如：发现 RM 喜欢长回答，就一直输出废话；发现 RM 喜欢某些词，就反复堆砌。

**KL 惩罚**限制了 PPO 训练后的模型不能偏离 SFT 模型太远：

$$r_{total}(x, y) = r_{RM}(x, y) - \beta \cdot KL\left[\pi_{RL}(y|x) \| \pi_{SFT}(y|x)\right]$$

- $\beta$ 越大 → 越保守，越像 SFT 模型
- $\beta$ 越小 → 越激进，越追求高分（但可能 reward hack）

## 4. 直观类比

| RLHF 组件 | 类比 |
|-----------|------|
| 预训练模型 | 刚毕业的实习生，知识丰富但不懂职场规则 |
| SFT | 入职培训，告诉他怎么写规范报告 |
| 奖励模型 | 有经验的老员工，会评价报告好不好 |
| PPO 优化 | 实习生根据老员工反馈不断改进 |
| KL 惩罚 | 不要改变太多，别变成完全不同的人 |

---

## 5. RLHF 的优缺点

### 优点
- 效果显著：InstructGPT（1.3B）表现优于原始 GPT-3（175B）
- 可以注入主观偏好（安全、礼貌、格式等）
- 泛化能力强

### 缺点
- **人工标注成本高**：需要大量人类排序数据
- **奖励模型不完美**：RM 本身也会出错，导致 reward hacking
- **训练不稳定**：PPO 超参数敏感，难以调优
- **慢**：PPO 需要多次前向/反向传播，显存占用大

---

## 6. 总结

```
预训练 LLM
    ↓ SFT（人工示范数据）
SFT 模型
    ↓ 人类排序数据 → 训练 RM
奖励模型
    ↓ PPO + KL 惩罚
对齐后的模型（InstructGPT / ChatGPT）
```

RLHF 是当前主流对齐方法的基础，PPO、DPO、GRPO 都是在解决它的不同缺点。

## 7. 关键词速查

| 术语 | 含义 |
|------|------|
| SFT | Supervised Fine-Tuning，监督微调 |
| RM | Reward Model，奖励模型 |
| PPO | Proximal Policy Optimization，近端策略优化 |
| KL 散度 | 衡量两个分布差异的指标 |
| Reward Hacking | 模型钻奖励模型的空子，得高分但实际质量差 |
| Alignment | 对齐，让模型行为符合人类意图 |
