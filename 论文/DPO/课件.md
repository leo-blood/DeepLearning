# DPO 课件：直接偏好优化

> 论文：*Direct Preference Optimization: Your Language Model is Secretly a Reward Model*（Rafailov et al., Stanford，2023）

---

## 1. DPO 要解决什么问题？

RLHF + PPO 效果好，但有三个痛点：

| 痛点 | 描述 |
|------|------|
| 复杂 | 需要单独训练奖励模型 RM |
| 慢 | PPO 需要 4 个模型同时在显存中 |
| 不稳定 | PPO 超参敏感，容易训练崩溃 |

**DPO 的答案**：不需要 RM，不需要 PPO，**直接用偏好数据微调模型**，数学上等价于 RLHF 的最优解。

---

## 2. 核心洞察：LLM 本身就是奖励模型

### 从 RLHF 推导

RLHF 的优化目标：

$$\max_{\pi} \mathbb{E}_{x \sim D, y \sim \pi} [r(x,y)] - \beta \cdot KL[\pi(y|x) \| \pi_{ref}(y|x)]$$

这个目标有**解析最优解**：

$$\pi^*(y|x) = \frac{1}{Z(x)} \pi_{ref}(y|x) \exp\left(\frac{1}{\beta} r(x,y)\right)$$

反过来推导，奖励可以用策略来表达：

$$r(x,y) = \beta \log \frac{\pi^*(y|x)}{\pi_{ref}(y|x)} + \beta \log Z(x)$$

**关键**：把这个奖励代入 Bradley-Terry 偏好模型（人类更偏好好回答的概率）：

$$p(y_w \succ y_l | x) = \sigma\left(\beta \log \frac{\pi(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi(y_l|x)}{\pi_{ref}(y_l|x)}\right)$$

$Z(x)$ 完全消掉了！

---

## 3. DPO 损失函数

$$\mathcal{L}_{DPO}(\pi_\theta) = -\mathbb{E}_{(x, y_w, y_l)} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right]$$

### 拆解理解

定义**隐式奖励差**：

$$\Delta = \underbrace{\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)}}_{\text{好回答的相对得分}} - \underbrace{\beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}}_{\text{差回答的相对得分}}$$

损失 $= -\log \sigma(\Delta)$

- $\Delta$ 越大（好回答得分明显高于差回答）→ 损失越小 → 训练目标满足

---

## 4. 直观图解

```
对于同一个 prompt x：

好回答 y_w：π_θ(y_w|x) / π_ref(y_w|x)  ← 希望这个比值变大（更喜欢好回答）
差回答 y_l：π_θ(y_l|x) / π_ref(y_l|x)  ← 希望这个比值变小（更不喜欢差回答）

DPO 梯度方向：
  ↑ 增大 log π_θ(y_w|x)（好回答概率上升）
  ↓ 减小 log π_θ(y_l|x)（差回答概率下降）
  β 控制偏离参考模型的程度
```

---

## 5. DPO 训练流程

```
数据：(prompt x, 好回答 y_w, 差回答 y_l) 三元组
       ↓
冻结 π_ref（SFT 模型，只做前向传播）
       ↓
训练 π_θ（初始化为 SFT 模型）：
  - 计算 π_θ(y_w|x) 和 π_θ(y_l|x)
  - 计算 π_ref(y_w|x) 和 π_ref(y_l|x)
  - 计算 DPO 损失，梯度下降
       ↓
完成！不需要 RM，不需要 PPO
```

### 对比 RLHF 流程

| 步骤 | RLHF | DPO |
|------|------|-----|
| 1 | SFT | SFT |
| 2 | 训练奖励模型 RM | ❌ 跳过 |
| 3 | PPO 在线采样优化 | 直接监督损失优化 |
| 模型数量 | 4 个 | 2 个（π_θ + π_ref） |

---

## 6. β 参数的含义

$$\beta \log \frac{\pi_\theta(y|x)}{\pi_{ref}(y|x)}$$

- $\beta$ 小 → 允许模型大幅偏离参考模型 → 更激进的偏好学习
- $\beta$ 大 → 保持接近参考模型 → 更保守
- 通常取 0.1 ~ 0.5

---

## 7. DPO 的优缺点

### 优点
- **简单**：标准的监督学习损失，像 SFT 一样训练
- **省资源**：只需要 2 个模型（训练模型 + 参考模型）
- **稳定**：没有 PPO 的复杂超参数
- **数学严格**：在 RLHF 假设下是最优解

### 缺点
- **离线方法（Offline）**：数据固定，不能在训练中探索新回答
- **数据质量敏感**：需要高质量的偏好对，噪声数据影响大
- **理论假设强**：假设 Bradley-Terry 模型，现实中人类偏好更复杂
- **实践效果有时不如 PPO**：在线采样的 PPO 能持续产生新数据分布

## 8. 代码示意

```python
def dpo_loss(pi_logps_chosen, pi_logps_rejected,
             ref_logps_chosen, ref_logps_rejected, beta=0.1):
    """
    pi_logps_chosen:   模型对好回答的 log 概率
    pi_logps_rejected: 模型对差回答的 log 概率
    ref_logps_*:       参考模型对应的 log 概率
    """
    # 计算对数比率差（隐式奖励差）
    pi_ratio  = pi_logps_chosen - pi_logps_rejected
    ref_ratio = ref_logps_chosen - ref_logps_rejected
    
    logits = beta * (pi_ratio - ref_ratio)
    
    # 等价于 -log(sigmoid(logits))
    loss = -F.logsigmoid(logits).mean()
    return loss
```

## 9. 直观类比

> 假设你在学写作：
> - **RLHF**：先找一个老师（RM）学会打分，再反复写文章让老师打分，根据分数修改
> - **DPO**：直接给你看两篇文章（好的 vs 差的），告诉你"向好的学习，远离差的"——不需要中间人

## 10. 关键词速查

| 术语 | 含义 |
|------|------|
| Preference Data | 偏好数据，$(x, y_w, y_l)$ 三元组 |
| Bradley-Terry 模型 | 用两个选项的分数差来建模人类偏好概率的模型 |
| Offline | 离线方法，数据固定，不在训练中采样新数据 |
| 隐式奖励 | DPO 中 LLM 本身隐含的奖励函数 |
| π_ref | 参考策略，即 SFT 模型（冻结参数） |
| β | 温度参数，控制偏离参考模型的程度 |
