① 用户请求流（同步）
  用户输入
    → API Gateway (JWT验证 + 限流)
    → Conversation Loop
        ├─ preflight: token >= 50% → 触发 Context 压缩
        ├─ 后台预取记忆(并发，不阻塞)
        ├─ plugin hooks 追加上下文到用户消息
        └─ LLM 调用(流式SSE返回)
    → Orchestrator 路由到对应 Agent
    → SSE 流式推送用户

② 自进化流（异步，不阻塞响应）
  对话结束
    → daemon thread 启动 Background Review Fork
    → 记忆模式: 写入 pgvector 长期记忆
    → 技能模式: 创建/修补 Skill 文件
    → Curator 空闲触发: 整理 Skill 生命周期

③ 知识库构建流
  文档上传(PDF/Word/HTML)
    → API Gateway
    → ARQ 异步队列
    → unstructured / MinerU 解析
    → 语义分块 → DeepSeek Embedding
    → Chroma 向量存储 + PostgreSQL 元信息

④ 知识库问答流
  用户提问
    → RAG Agent
    → Redis 语义缓存命中? → 直接返回
    → Chroma 向量检索(HNSW索引)
    → Top-K chunks 注入 context
    → DeepSeek 生成回答 + 来源引用

⑤ LLM Fallback 流
  DeepSeek 429 / 503
    → 自动切换 Qwen-Plus
    → 重置 retry 计数
    → 无感知继续对话
    → 恢复后自动切回 DeepSeek

⑥ Context 压缩流
  token >= 50% 阈值
    → Step1(廉价): 历史工具结果 → 单行摘要, 剥离历史图片
    → Step2(LLM): 保护头部+尾部(20K), 中间段 LLM 摘要
    → 增量更新(不重复压缩已压缩部分)
    → 防抖: 连续2次节省 < 10% 则停止

⑦ 危险命令审批流
  工具调用检测到高危操作(rm -rf / DROP TABLE 等)
    → 普通用户会话: WebSocket 推送审批请求 → 等待前端确认
    → Cron / 子 Agent:  自动拒绝，记录日志

⑧ 子 Agent 并行流
  Orchestrator 拆分独立子任务
    → asyncio.gather() 并发派发多个 Agent
    → 每个子 Agent: 独立工具集 + 禁止嵌套派发(depth=1)
    → 结果汇聚 → Orchestrator 整合返回


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  技术选型速查  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  LLM          DeepSeek (主) · Qwen-Plus (fallback)
  Embedding    DeepSeek-embedding
  RAG          LlamaIndex · Chroma (当前) → Milvus (扩展)
  文档解析      unstructured (通用) · MinerU (PDF专项)
  代码沙箱      E2B (托管, 无需自建Docker)
  浏览器工具    Playwright
  后端          FastAPI + Uvicorn
  前端          Next.js + Tailwind CSS
  数据库        PostgreSQL + pgvector
  缓存          Redis
  异步队列      ARQ / Celery
  文件存储      OSS / 本地磁盘
  可观测性      OpenTelemetry + 结构化 JSON 日志
  部署          Docker Compose
