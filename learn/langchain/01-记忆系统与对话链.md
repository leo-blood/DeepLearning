## 一、为什么需要 Memory？

LLM 本身是**无状态**的——每次调用都是全新对话，不记得上一轮说了什么。

```
用户：我叫张三
LLM ：你好，张三！

用户：我叫什么？
LLM ：[无法回答，因为它不记得第一轮了]
```

Memory 的本质是：**在每次调用前，将历史对话注入 prompt；调用后，将新内容存回 Memory。**

```
┌─────────────────────────────────────────────┐
│                Memory 工作流                  │
│                                             │
│  用户输入 ──► load_memory_variables()        │
│               ↓                             │
│           组装 Prompt（含历史）               │
│               ↓                             │
│           调用 LLM                           │
│               ↓                             │
│           save_context(input, output)        │
│               ↓                             │
│           存入 Memory Store                  │
└─────────────────────────────────────────────┘
```

## 二、四种内置 Memory 对比

### 2.1 ConversationBufferMemory（全量缓冲）

把所有历史消息原文保存，原样注入。

```python
from langchain.memory import ConversationBufferMemory
from langchain_openai import ChatOpenAI
from langchain.chains import ConversationChain

memory = ConversationBufferMemory(
    return_messages=True,   # 返回 HumanMessage/AIMessage 对象而非字符串
    memory_key="history",   # 注入 prompt 时用的变量名
)

chain = ConversationChain(
    llm=ChatOpenAI(model="gpt-4o"),
    memory=memory,
    verbose=True,           # 打印完整 prompt，调试用
)

chain.invoke({"input": "我叫张三，我是一名后端工程师"})
chain.invoke({"input": "我的职业是什么？"})
# ✅ 能正确回答：后端工程师
```

**优点**：信息完整，实现简单  
**缺点**：随对话增长，token 数无限膨胀，长对话会超出 context window

**适用场景**：短对话（< 20 轮）、测试原型

---

### 2.2 ConversationWindowMemory（滑动窗口）

只保留最近 k 轮对话。

```python
from langchain.memory import ConversationWindowMemory

memory = ConversationWindowMemory(
    k=5,                    # 只保留最近 5 轮
    return_messages=True,
)
```

**适用场景**：只需要近期上下文的任务（如代码补全、简短问答）

---

### 2.3 ConversationSummaryMemory（摘要压缩）

超出阈值后，让 LLM 将旧对话压缩成摘要，节省 token。

```python
from langchain.memory import ConversationSummaryMemory

memory = ConversationSummaryMemory(
    llm=ChatOpenAI(model="gpt-4o-mini"),   # 用小模型做摘要，省钱
    return_messages=True,
)
```

**工作原理**：

```
轮次 1-10：原文保存
轮次 11：[摘要] "用户张三是后端工程师，讨论了数据库优化..." + 轮次 9-10 原文
轮次 12：[摘要更新] + 轮次 10-11 原文
```

**适用场景**：长对话、用户信息需要持久保留

---

### 2.4 ConversationSummaryBufferMemory（混合策略，推荐）

结合摘要 + 缓冲：近期消息保留原文，超出 token 阈值后自动摘要旧内容。

```python
from langchain.memory import ConversationSummaryBufferMemory

memory = ConversationSummaryBufferMemory(
    llm=ChatOpenAI(model="gpt-4o-mini"),
    max_token_limit=2000,   # 超过此 token 数时触发摘要
    return_messages=True,
)
```

**这是生产环境最常用的 Memory 类型。**

---

### 2.5 四种类型对比表

| Memory 类型 | Token 增长 | 信息完整性 | 适用场景 |
|-------------|-----------|-----------|---------|
| Buffer | O(n) 线性增长 | ★★★★★ | 短对话原型 |
| Window | O(1) 固定 | ★★☆☆☆ | 只需近期上下文 |
| Summary | O(摘要大小) | ★★★☆☆ | 长对话，关注全局信息 |
| SummaryBuffer | O(摘要+窗口) | ★★★★☆ | **生产推荐** |

## 三、Memory 与 LCEL 集成（现代用法）

LangChain 新版推荐用 LCEL + `RunnableWithMessageHistory` 替代旧版 Chain + Memory。

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_community.chat_message_histories import ChatMessageHistory
from langchain_openai import ChatOpenAI

# 1. 定义 prompt（含历史占位符）
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个专业的技术助手。"),
    MessagesPlaceholder(variable_name="history"),  # 历史消息插入点
    ("human", "{input}"),
])

# 2. 构建 chain
llm = ChatOpenAI(model="gpt-4o")
chain = prompt | llm

# 3. 内存存储（生产中替换为 Redis）
store = {}

def get_session_history(session_id: str) -> ChatMessageHistory:
    if session_id not in store:
        store[session_id] = ChatMessageHistory()
    return store[session_id]

# 4. 包装为带历史的 Runnable
chain_with_history = RunnableWithMessageHistory(
    chain,
    get_session_history,
    input_messages_key="input",
    history_messages_key="history",
)

# 5. 多轮对话
config = {"configurable": {"session_id": "user_123"}}

chain_with_history.invoke({"input": "我叫张三，我是后端工程师"}, config=config)
chain_with_history.invoke({"input": "我的职业是什么？"}, config=config)
```

## 四、持久化 Memory 到 Redis

```python
from langchain_community.chat_message_histories import RedisChatMessageHistory

def get_session_history(session_id: str) -> RedisChatMessageHistory:
    return RedisChatMessageHistory(
        session_id=session_id,
        url="redis://localhost:6379",
        ttl=3600,          # 1小时过期
        key_prefix="chat:",
    )

chain_with_history = RunnableWithMessageHistory(
    chain,
    get_session_history,
    input_messages_key="input",
    history_messages_key="history",
)
```

**Redis 中的存储结构：**

```
chat:user_123  →  List [
    {"type": "human", "content": "我叫张三"},
    {"type": "ai",    "content": "你好，张三！"},
    ...
]
```

## 五、自定义 Memory：注入结构化业务上下文

有时候需要把用户信息（来自数据库）直接注入对话，而不是从历史消息里推断。

```python
from langchain_core.memory import BaseMemory
from langchain_core.messages import SystemMessage
from typing import Dict, Any, List

class UserProfileMemory(BaseMemory):
    """将用户档案作为 System 消息注入"""
    user_profile: Dict[str, Any] = {}
    memory_key: str = "history"

    @property
    def memory_variables(self) -> List[str]:
        return [self.memory_key]

    def load_memory_variables(self, inputs: Dict[str, Any]) -> Dict[str, Any]:
        profile_text = "\n".join(
            f"- {k}: {v}" for k, v in self.user_profile.items()
        )
        return {
            self.memory_key: [
                SystemMessage(content=f"用户档案：\n{profile_text}")
            ]
        }

    def save_context(self, inputs: Dict[str, Any], outputs: Dict[str, str]) -> None:
        pass  # 档案由外部系统维护，不需要从对话中更新

    def clear(self) -> None:
        self.user_profile = {}


# 使用
memory = UserProfileMemory(
    user_profile={
        "姓名": "张三",
        "职业": "后端工程师",
        "技术栈": "Java / Python",
        "当前项目": "分布式缓存优化",
    }
)
```

## 六、常见陷阱

### 陷阱 1：memory_key 不匹配

```python
# ❌ 错误：memory_key 与 prompt 变量名不一致
memory = ConversationBufferMemory(memory_key="chat_history")
prompt = ChatPromptTemplate.from_messages([
    MessagesPlaceholder(variable_name="history"),  # 名字对不上！
])

# ✅ 正确：保持一致
memory = ConversationBufferMemory(memory_key="history")
```

### 陷阱 2：忘记 return_messages=True

```python
# ❌ 返回字符串格式，与 ChatPromptTemplate 不兼容
memory = ConversationBufferMemory()
# memory.load_memory_variables({}) → {"history": "Human: ...\nAI: ..."}

# ✅ 返回消息对象列表
memory = ConversationBufferMemory(return_messages=True)
# memory.load_memory_variables({}) → {"history": [HumanMessage(...), AIMessage(...)]}
```

### 陷阱 3：多用户共享同一 Memory 实例

```python
# ❌ 危险：多个用户的对话混在一起
shared_memory = ConversationBufferMemory()

# ✅ 正确：每个 session_id 独立
store: Dict[str, ChatMessageHistory] = {}
def get_session_history(session_id: str):
    if session_id not in store:
        store[session_id] = ChatMessageHistory()
    return store[session_id]
```
