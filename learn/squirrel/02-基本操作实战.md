## 1. 操作方法全览

### 1.1 SquirrelOpService 方法列表

```java
// 本项目封装的 Squirrel 操作类
public class SquirrelOpService {

    // ===== 基本读写 =====
    <T> T getString(String key)                              // 读取值
    Boolean setString(String key, Object value, int expire)  // 写入，带过期

    Long decrBy(String key, long amount, int expire)             // 原子减
}
```

## 2. 读写操作

### 2.1 写入（setString）

```java
// 写入字符串
squirrelOpService.setString("user:zhangsan", "online", 300);
// Redis: SET cratos-supplement:user:zhangsan "online" EX 300

// 写入对象（框架自动序列化）
StaffDTO staffDTO = new StaffDTO("zhangsan", "张三");
squirrelOpService.setString("staff:zhangsan", staffDTO, 600);
// Redis: SET cratos-supplement:staff:zhangsan <序列化后的对象> EX 600

// 写入 Map
Map<String, Integer> config = new HashMap<>();
config.put("pageSize", 200);
squirrelOpService.setString("job:config", config, 86400);
```

**过期时间参数（单位：秒）**：

```java
// 常用过期时间
60          // 1分钟
300         // 5分钟
3600        // 1小时
86400       // 1天
60 * 30     // 30分钟（本项目锁的过期时间）
```

### 2.2 读取（getString）

```java
// 读取字符串
String status = squirrelOpService.getString("user:zhangsan");
// status 可能为 null（key 不存在或已过期）

// 读取对象（需要强制转型）
StaffDTO staff = squirrelOpService.getString("staff:zhangsan");
// 注意：如果 key 不存在，返回 null

// 安全读取（防止 NullPointerException）
String value = squirrelOpService.getString("some:key");
if (value != null) {
    // 使用 value
}
```

### 2.3 删除（delString）

```java
// 删除 key
Boolean deleted = squirrelOpService.delString("user:zhangsan");
// deleted=true：删除成功
// deleted=false：key 不存在

// 典型场景：释放分布式锁
squirrelOpService.delString(String.format("staff_id_lock-%s", staffId));
```

## 3. 缓存模式实战

### 3.1 Cache-Aside（旁路缓存）模式

```java
// 标准缓存用法
public StaffDTO getStaffInfo(String mis) {
    String cacheKey = "staff_info:" + mis;
    int cacheSeconds = 300;    // 缓存5分钟

    // 1. 先查缓存
    StaffDTO cached = squirrelOpService.getString(cacheKey);
    if (cached != null) {
        log.debug("缓存命中, mis:{}", mis);
        return cached;
    }

    // 2. 缓存未命中，查数据源（Pigeon / DB）
    StaffDTO fresh = staffPigeonService.queryStaffDTOByStaffMis(mis);
    if (fresh == null) {
        return null;
    }

    // 3. 写入缓存
    try {
        squirrelOpService.setString(cacheKey, fresh, cacheSeconds);
    } catch (Exception e) {
        // 缓存写入失败不影响业务，只打日志
        log.warn("缓存写入失败, mis:{}", mis, e);
    }

    return fresh;
}
```

### 3.2 缓存失效与更新

```java
// 当员工信息更新时，主动删除缓存（不等自然过期）
public void updateStaffInfo(String mis, StaffDTO newInfo) {
    // 1. 更新数据库
    staffMapper.update(newInfo);

    // 2. 删除缓存（下次读取时自动从 DB 重建缓存）
    String cacheKey = "staff_info:" + mis;
    try {
        squirrelOpService.delString(cacheKey);
        log.info("缓存已删除, mis:{}", mis);
    } catch (Exception e) {
        log.warn("缓存删除失败（接受），key:{}", cacheKey, e);
        // 不影响业务，缓存会在过期后自动失效
    }
}
```

## 4. 原子计数器

### 4.1 incrBy 详解

```java
// incrBy 的语义：
// key 存在 → 原子加 amount，返回新值
// key 不存在 → 创建 key，初始值 = amount，返回 amount

Long count = squirrelOpService.incrBy("task:exec:count", 1L, 86400);
// count = 1（第一次调用）
// count = 2（第二次调用）
// ...
```

### 4.2 任务执行次数统计

```java
// 统计今天某个任务执行了多少次
public void recordJobExecution(String jobName) {
    String today = new SimpleDateFormat("yyyyMMdd").format(new Date());
    String counterKey = String.format("job:exec:%s:%s", jobName, today);

    try {
        Long count = squirrelOpService.incrBy(counterKey, 1L, 86400);  // 保留1天
        if (count != null && count > 100) {
            // 今天执行超过100次，发告警
            log.warn("任务执行次数异常, jobName:{}, count:{}", jobName, count);
        }
    } catch (Exception e) {
        log.warn("计数失败（可接受），jobName:{}", jobName, e);
    }
}
```

### 4.3 限流计数器

```java
// 接口限流：同一员工1分钟内最多提交5次
public boolean checkRateLimit(String staffMis) {
    String minute = new SimpleDateFormat("yyyyMMddHHmm").format(new Date());
    String limitKey = String.format("rate_limit:%s:%s", staffMis, minute);

    try {
        Long count = squirrelOpService.incrBy(limitKey, 1L, 70);   // 70秒过期（稍大于1分钟）
        if (count != null && count > 5) {
            log.warn("访问频率超限, staffMis:{}, count:{}", staffMis, count);
            return false;   // 超限，拒绝
        }
        return true;        // 允许
    } catch (Exception e) {
        log.warn("限流检查失败（放行）, staffMis:{}", staffMis, e);
        return true;        // 异常时放行（Fail-Open 策略）
    }
}
```

### 4.4 decrBy 扣减场景

```java
// 活动名额扣减（库存）
// 假设 "activity:quota:123" = 100（初始名额100）
public boolean deductQuota(Long activityId) {
    String quotaKey = "activity:quota:" + activityId;

    try {
        Long remaining = squirrelOpService.decrBy(quotaKey, 1L, 86400);
        if (remaining != null && remaining < 0) {
            // 已经扣到负数，名额用完，回补
            squirrelOpService.incrBy(quotaKey, 1L, 86400);
            return false;
        }
        return true;
    } catch (Exception e) {
        log.error("名额扣减失败, activityId:{}", activityId, e);
        return false;
    }
}
```

## 5. 异常处理规范

### 5.1 Squirrel 的两类异常

```java
// Squirrel 可能抛出两个不同包的异常：
com.dianping.squirrel.asyncclient.exception.StoreException
com.dianping.squirrel.common.exception.StoreException

// SquirrelOpService 已处理异常：
// - setNtxString：catch 后返回 false
// - getString/setString：方法签名 throws StoreException，调用方需处理
```

### 5.2 缓存操作异常处理原则

```
原则：缓存异常不能影响主业务！

缓存读：
  try { return cache.get(key) } catch { return null }  → 降级到数据库查询

缓存写：
  try { cache.set(key, value) } catch { log.warn(...) }  → 忽略，下次读时再写

删缓存：
  try { cache.del(key) } catch { log.warn(...) }  → 忽略，等自然过期

分布式锁（例外）：
  加锁失败 → 返回 false，不抛异常（SquirrelOpService 已处理）
  业务层判断 false → 直接返回"请勿重复提交"
```

### 5.3 完整的带异常处理的缓存代码

```java
public StaffLevel getStaffLevel(String mis) {
    String cacheKey = "staff_level:" + mis;

    // 读缓存（异常降级）
    StaffLevel cached = null;
    try {
        cached = squirrelOpService.getString(cacheKey);
    } catch (StoreException e) {
        log.warn("读取等级缓存失败（降级查DB），mis:{}", mis, e);
    }
    if (cached != null) return cached;

    // 查数据库
    StaffLevel level = staffLevelMapper.queryByMis(mis);
    if (level == null) return null;

    // 写缓存（异常不影响返回）
    try {
        squirrelOpService.setString(cacheKey, level, 300);
    } catch (StoreException e) {
        log.warn("写入等级缓存失败（可接受），mis:{}", mis, e);
    }

    return level;
}
```

## 6. Key 设计规范

### 6.1 命名格式

```
{业务}:{实体}:{ID/维度}

示例：
  staff_id_lock-{staffId}          分布式锁（本项目实际使用）
  staff_info:{mis}                 员工信息缓存
  task_exec_count:{taskId}:{date}  任务执行计数
  rate_limit:{mis}:{minute}        限流计数
  activity_quota:{activityId}      活动名额
```

### 6.2 key 过期时间选择参考

```java
// 锁：业务最长执行时间的 2倍（防死锁）
int LOCK_EXPIRE = 60 * 30;   // 30分钟

// 员工信息缓存：5分钟（变化不频繁，短时不一致可接受）
int STAFF_CACHE = 300;

// 计数器：统计周期 + 10%（日维度就是 86400 秒）
int DAILY_COUNTER = 86400;

// 限流：滑动窗口大小（分钟级别就是 70 秒）
int RATE_LIMIT = 70;

// 会话状态：半小时（与用户 session 同步）
int SESSION = 1800;
```
