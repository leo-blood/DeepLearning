## 1. Producer 配置详解

### 1.1 mafka.properties 中定义 Producer

每个 Producer 对应 mafka.properties 中一段配置：

```properties
# Producer 索引从 [0] 开始，可以有任意多个

mdp.mafka.producer[1].producerName=delayAssignIncentiveTaskProducer
#                    ↑ Spring Bean 名称，代码中通过 @Qualifier 注入

mdp.mafka.producer[1].bgNameSpace=pingtai
#                    ↑ 消息集群命名空间（必须与消费者一致）

mdp.mafka.producer[1].appkey=com.sankuai.csccratos.ccsadmin
#                    ↑ 发消息的应用标识（用于监控和权限）

mdp.mafka.producer[1].topicName=ccsadmin_delay_issue_tasks
#                    ↑ 消息主题

mdp.mafka.producer[1].delay=true
#                    ↑ 是否开启延迟消息支持（只有 true 时才能调 sendDelayMessage）
```

### 1.2 本项目 6 个 Producer 配置


## 2. Producer Bean 注入

### 2.1 @Qualifier 注入

MDP 根据 `producerName` 创建对应的 `IProducerProcessor` Bean：

```java
@Service
public class DelayDoIncentiveTaskProducer {

    // 通过 @Qualifier 指定要注入哪个 Producer Bean
    @Autowired
    @Qualifier("delayAssignIncentiveTaskProducer")   // 与 producerName 完全一致
    private IProducerProcessor producer;
}
```

### 2.2 如果 Bean 名写错了

```java
// ❌ 错误：Bean 名不存在
@Qualifier("delayAssignIncentiveTask")   // 少了 Producer
private IProducerProcessor producer;
// 启动报错：NoSuchBeanDefinitionException

// ❌ 错误：大小写不匹配
@Qualifier("DelayAssignIncentiveTaskProducer")   // 首字母大写
// 启动报错：NoSuchBeanDefinitionException
```

## 3. 发送普通消息

### 3.1 基础发送

```java
@Service
@Slf4j
public class CcsStaffOffDutyProducer {

    @Autowired
    @Qualifier("ccsOrgProcessProducer")
    private IProducerProcessor producer;

    public void sendOffDutyMessage(String staffMis, String eventType) {
        // 构建消息体（JSON 字符串）
        Map<String, Object> msgMap = new HashMap<>();
        msgMap.put("staffMis", staffMis);
        msgMap.put("eventType", eventType);
        msgMap.put("timestamp", System.currentTimeMillis());
        String msgBody = JSONUtil.toJSONString(msgMap);

        try {
            // 发送消息
            ProducerResult result = producer.sendMessage(msgBody);
            log.info("发送离职消息成功, staffMis:{}, result:{}", staffMis, result);
        } catch (Exception e) {
            log.error("发送离职消息失败, staffMis:{}", staffMis, e);
            // 注意：发送失败不一定要抛异常，根据业务决定是否重试
        }
    }
}
```

### 3.2 ProducerResult 处理

```java
ProducerResult result = producer.sendMessage(msgBody);

// ProducerResult 包含：
//   isSuccess()    是否发送成功
//   getMsgId()     消息ID（用于追踪）
//   getErrorMsg()  失败原因

if (!result.isSuccess()) {
    log.error("消息发送失败, msgId:{}, error:{}", result.getMsgId(), result.getErrorMsg());
    // 决策：是否需要重试或告警
}
```

## 4. 发送延迟消息（核心特性）

### 4.1 延迟消息原理

```
sendDelayMessage(msgBody, delayMs)
  │
  ▼
Mafka 接收消息，标记延迟时间
  │
  │（等待 delayMs 毫秒）
  │
  ▼
到达投递时间，消费者才能看到此消息
```

### 4.2 延迟圈人实现

```java
// DelayDoIncentiveTaskProducer.java
@Service
@Slf4j
public class DelayDoIncentiveTaskProducer {

    @Autowired
    @Qualifier("delayAssignIncentiveTaskProducer")
    private IProducerProcessor producer;

    /**
     * 在任务创建时投递延迟消息，等任务开始时间到了再执行圈人
     */
    public void distributeIncentiveTaskByDelayMQ(IncentiveTask incentiveTask) {
        // 1. 构建消息体（只包含 ID 和版本号，Consumer 收到后再查完整数据）
        IncentiveDistributeTaskInfoBO bo = new IncentiveDistributeTaskInfoBO();
        bo.setTaskId(incentiveTask.getId());
        bo.setTaskVersion(incentiveTask.getVersion());   // 用于幂等校验
        String msgBody = JSONUtil.toJSONString(bo);

        // 2. 计算延迟时间：任务开始时间 - 当前时间
        Date startDate = incentiveTask.getTaskStartDate();
        long remainingTime = startDate.getTime() - System.currentTimeMillis();
        long delayTime = remainingTime > 5000 ? remainingTime : 5000;  // 最少延迟5秒

        try {
            // 3. 发送延迟消息（必须是 delay=true 的 Producer）
            ProducerResult result = producer.sendDelayMessage(msgBody, delayTime);
            log.info("延迟圈人消息投递成功, taskId:{}, delay:{}ms, result:{}",
                bo.getTaskId(), delayTime, result);
        } catch (Exception e) {
            log.error("延迟圈人消息投递失败, taskId:{}", bo.getTaskId(), e);
        }
    }
}
```

### 4.3 普通 send vs 延迟 send

```java
// 立即投递（Consumer 马上能消费）
producer.sendMessage(msgBody);

// 延迟投递（5分钟后才能消费）
producer.sendDelayMessage(msgBody, 5 * 60 * 1000L);

// 延迟投递到指定时间（明天早上8点）
long tomorrowMorning = ...;
long delayMs = tomorrowMorning - System.currentTimeMillis();
producer.sendDelayMessage(msgBody, delayMs);
```

**注意**：只有 `delay=true` 的 Producer 才支持 `sendDelayMessage`，否则报错。

## 5. 消息体设计最佳实践

### 5.1 瘦消息 vs 胖消息

```java
// 瘦消息（推荐）：只带 ID 和版本
{
    "taskId": 12345,
    "taskVersion": 3       ← 版本号用于幂等校验
}
// 优点：消息小，Consumer 收到后用 ID 查最新数据（避免数据时效问题）
// 缺点：Consumer 需要查库

// 胖消息：带全量数据
{
    "taskId": 12345,
    "taskName": "...",
    "staffList": [...500个mis...],   ← 消息可能很大！
    "config": {...}
}
// 优点：Consumer 不用查库
// 缺点：消息大，数据可能已过时（消息发出后数据被修改了）
```

本项目采用瘦消息策略：消息只含 `taskId` + `taskVersion`，Consumer 收到后重新查库。

### 5.2 版本号的重要性

```java
// 场景：任务创建后被修改了
// T=0：创建任务，version=1，投递消息 {taskId:123, taskVersion:1}
// T=1：任务被修改，version=2
// T=2：Consumer 收到消息

// Consumer 处理：
IncentiveTask task = taskMapper.queryById(bo.getTaskId());
if (!bo.getTaskVersion().equals(task.getVersion())) {
    // 版本不匹配，说明任务已被更新
    // 应该有新消息（version=2）会到来，这条消息跳过
    log.info("版本不匹配，跳过 taskId:{}, msgVersion:{}, dbVersion:{}",
        bo.getTaskId(), bo.getTaskVersion(), task.getVersion());
    return ConsumeStatus.CONSUME_SUCCESS;
}
// 版本匹配，正常处理
```

## 6. 发送失败的处理策略

### 6.1 核心激励任务（必须可靠）

```java
// 核心业务：发送失败需要降级处理
public void sendCriticalMessage(Long taskId) {
    try {
        ProducerResult result = producer.sendMessage(msgBody);
        if (!result.isSuccess()) {
            // MQ 发送失败，降级：写入数据库待补偿表
            failedMessageMapper.insert(new FailedMessage(taskId, msgBody));
            log.error("消息发送失败，已写入补偿表, taskId:{}", taskId);
        }
    } catch (Exception e) {
        failedMessageMapper.insert(new FailedMessage(taskId, msgBody));
        log.error("消息发送异常，已写入补偿表, taskId:{}", taskId, e);
    }
}
```

### 6.2 非核心通知（可以丢弃）

```java
// 非关键通知：发送失败只打日志
public void sendNotification(String staffMis, String msg) {
    try {
        producer.sendMessage(buildMsg(staffMis, msg));
    } catch (Exception e) {
        // 只打日志，不影响主流程
        log.warn("通知消息发送失败（可接受）, staffMis:{}", staffMis, e);
    }
}
```
