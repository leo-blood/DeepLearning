# Lion 基础概念与入门

---

## 1. 配置管理的演进

### 1.1 阶段一：硬编码（最原始）

```java
// 直接写在代码里
private static final int PAGE_SIZE = 200;
private static final String ERROR_MSG = "系统繁忙，请稍后重试";
```

问题：修改配置 = 改代码 + 重新编译 + 重新部署。

### 1.2 阶段二：配置文件

```properties
# application.properties
page.size=200
error.message=系统繁忙，请稍后重试
```

改进：配置与代码分离，部署时可替换文件。
问题：**还是需要重启服务**，修改文案要走完整发布流程（10-30分钟）。

### 1.3 阶段三：配置中心（Lion）

```java
@MdpConfig("page.size:200")       // Lion 控制台有值就用，否则用默认值 200
private Integer pageSize;

@MdpConfig("error.message:系统繁忙，请稍后重试")
private String errorMessage;
```

**改进**：在 Lion 控制台修改配置值 → **秒级推送** → **不重启服务生效**。

## 2. Lion 是什么

### 2.1 Lion 的定位

Lion 是美团内部配置中心，类比开源的 Apollo/Nacos Config：

```
Lion 解决的问题：
  ✅ 配置动态更新（不重启）
  ✅ 环境隔离（prod/staging/dev 各自独立）
  ✅ 权限管控（谁能改哪个配置）
  ✅ 变更审计（谁在什么时间改了什么）
  ✅ 灰度推送（只推给指定机器）
  ✅ 配置回滚（改错了一键还原）
```

### 2.2 Lion 与 MDP 的集成

MDP 把 Lion 接入封装为 `@MdpConfig` 注解，启动时自动：
1. 拉取该应用（AppKey）下的所有配置
2. 注入到 `@MdpConfig` 标注的字段
3. 监听 Lion 变更推送，字段实时更新

## 3. @MdpConfig 注解入门

### 3.1 最简用法

```java
import com.meituan.mdp.boot.starter.config.annotation.MdpConfig;

@Component
public class LionService {

    // 基本格式：@MdpConfig("配置key")
    @MdpConfig("dx.token")
    public String dxToken;       // 大象 token，从 Lion 读取

    // 带默认值：@MdpConfig("配置key:默认值")
    @MdpConfig("incentive.room.size:50")
    public Integer roomSize;     // Lion 未配置时默认 50
}
```

### 3.2 默认值的两种写法

```java
// 写法1：注解内默认值（推荐，语义清晰）
@MdpConfig("page.size:200")
private Integer pageSize;

// 写法2：字段初始值（Lion 读取失败时的兜底）
@MdpConfig("page.size")
private Integer pageSize = 200;

// 两者同时存在时，注解内默认值优先
// 规则：Lion有值 > 注解内默认值 > 字段初始值
```

### 3.3 配置 key 的命名规范

本项目的命名规律（无严格规范，但有惯例）：

```
全小写+点分隔（通用配置）：
  dx.token
  incentive.room.size
  ccs.mis.message.size

全大写+下划线（功能开关/文案）：
  CLOUD_OFFICE_APPLY_GRAY
  CLOUD_OFFICE_APPLY_LEADER_SUCCESS
  CLOUD_OFFICE_APPLY_SUCCESS_STAFF_DAYS_BEFORE

混合：
  cancel.staff.class.to.stag.msg
  org.formal.skill.ids
```

## 4. Lion 配置的生命周期

### 4.1 启动时初始化

```
应用启动
  │
  ├── MDP 检测 @MdpConfig 注解的字段
  ├── 向 Lion 服务端请求该 AppKey 的所有配置
  ├── Lion 返回配置 key-value 列表
  ├── 框架通过反射将值注入到对应字段
  │
  ▼
字段有值，可以正常使用
```

### 4.2 运行时动态更新

```
Lion 控制台修改配置值
  │
  ├── Lion 服务端检测到变更
  ├── 向订阅该配置的应用推送变更通知
  ├── MDP 接收到推送
  ├── 通过反射更新对应字段
  │
  ▼
字段已更新为新值（无需重启）

时延：通常 1~5 秒内生效
```

### 4.3 Lion 不可用时的降级

```
Lion 服务端不可用（极罕见）：
  ├── 应用启动时：使用注解内默认值 / 字段初始值（不影响启动）
  └── 运行时：字段保持上次成功拉取的值不变（不影响运行）
```

## 5. 本项目 LionService 设计

本项目将所有 Lion 配置**集中管理**在 `LionService.java`：

```java
// ccsadmin-infrastructure/proxy/LionService.java
@Component
@Data
public class LionService {

    // ===== 大象消息相关 =====
    @MdpConfig("dx.token")
    public String DX_TOKEN;

    // ===== 激励任务相关 =====
    @MdpConfig("default.incentive.delay.minutes")
    public Integer incentiveDelayMinutes;

    @MdpConfig("incentive.ess.inprogress.url")
    public String essInProgressUrl;

    // ===== 技能相关 =====
    @MdpConfig("default.es.staff.skill.amount")
    public Integer esStaffSkillAmount = 1000;

    @MdpConfig("org.formal.skill.ids")
    public String[] formalSkillIds;

    // ===== 指标管控相关 =====
    @MdpConfig("ccs.metric.controller.condition")
    public HashMap<Integer, Integer> metricControllerMap = Maps.newHashMap();

    // ... 20+ 个配置项
}
```

**为什么集中管理**：
- 一处查看所有动态配置，避免配置分散难以维护
- 多个业务类复用同一个 `LionService` Bean
- 统一管理默认值和类型转换

**例外**：只在某一个类内使用的配置，可以直接在业务类中使用：

```java
// CloudOfficeStaffApplyService.java
// 这些文案只有申请服务用，直接写在业务类
@MdpConfig("CLOUD_OFFICE_APPLY_LEADER_SUCCESS:您管理的员工...")
private String CLOUD_OFFICE_APPLY_LEADER_SUCCESS;
```

## 6. Lion vs 其他配置方式

| 配置方式 | 修改需重启 | 环境隔离 | 动态推送 | 权限管控 | 审计日志 |
|---------|-----------|---------|---------|---------|---------|
| 硬编码 | ✅（改代码）| ❌ | ❌ | ❌ | ❌ |
| application.properties | ✅（重启）| ✅ | ❌ | ❌ | ❌ |
| 数据库配置表 | ❌ | ✅ | 轮询 | 需自建 | 需自建 |
| **Lion** | ❌ | ✅ | ✅（秒级）| ✅ | ✅ |

## 7. 什么应该放 Lion，什么不该放

```
✅ 适合放 Lion：
  功能开关：CLOUD_OFFICE_APPLY_GRAY（灰度控制）
  阈值参数：incentive.room.size, esStaffSkillAmount
  运营文案：错误提示语、消息模板
  跳转 URL：ESS任务链接（随时可能变）
  批量大小：每批处理多少条（线上调参）

❌ 不适合放 Lion：
  密钥/Token → 放 KMS（Lion 控制台权限较宽）
  数据库地址 → 放 application.properties（启动时需要）
  业务数据 → 放数据库（不是配置，是数据）
  固定常量 → 放代码常量（永远不变的值）
```
