# Lion 配置类型详解

> 本文聚焦：@MdpConfig 支持哪些数据类型，各自怎么配置和使用。

---

## 1. 基本类型

### 1.1 String 类型

```java
@MdpConfig("dx.token")
public String DX_TOKEN;                    // Lion 未配置时为 null

@MdpConfig("error.msg:系统繁忙，请稍后重试")
private String errorMsg;                   // 有默认值

// Lion 控制台配置：
// key: dx.token
// value: eyJhbGciOiJSUzI1NiJ9...（token字符串）
```

### 1.2 Integer / Long / Double

```java
@MdpConfig("incentive.room.size:50")
public Integer roomSize;                   // 整数，默认50

@MdpConfig("default.es.staff.skill.amount")
public Integer esStaffSkillAmount = 1000;  // 字段默认值

@MdpConfig("task.timeout.ms:5000")
private Long taskTimeoutMs;

@MdpConfig("score.threshold:0.85")
private Double scoreThreshold;

// Lion 控制台配置：
// key: incentive.room.size
// value: 100
// → 框架自动将 "100" 转换为 Integer 100
```

### 1.3 Boolean

```java
@MdpConfig("feature.new.logic.enabled:false")
private Boolean newLogicEnabled;

// Lion 控制台：
// value: true  → Boolean.TRUE
// value: false → Boolean.FALSE
// value: 1     → 注意：Boolean 不识别 "1"，必须用 "true"/"false"

// 使用
if (Boolean.TRUE.equals(newLogicEnabled)) {
    // 新逻辑
} else {
    // 老逻辑
}
```

## 2. 数组类型

### 2.1 String[] 数组

```java
// LionService.java
@MdpConfig("org.formal.skill.ids")
public String[] formalSkillIds;

@MdpConfig("org.default.paymentManagementMis.list")
public String[] defaultPaymentManagementMisList;

// Lion 控制台配置（逗号分隔）：
// key: org.formal.skill.ids
// value: skill001,skill002,skill003

// 注入后：["skill001", "skill002", "skill003"]
```

### 2.2 数组的使用

```java
// 检查技能是否在正式技能组列表中
public boolean isFormalSkill(String skillId) {
    if (lionService.formalSkillIds == null || lionService.formalSkillIds.length == 0) {
        return false;
    }
    return Arrays.asList(lionService.formalSkillIds).contains(skillId);
}

// 获取支付管理人员列表
public List<String> getPaymentManagers() {
    if (lionService.defaultPaymentManagementMisList == null) {
        return Collections.emptyList();
    }
    return Arrays.asList(lionService.defaultPaymentManagementMisList);
}
```

### 2.3 注意事项

```
✅ 正确的 Lion 配置值：skill001,skill002,skill003
   注入结果：["skill001", "skill002", "skill003"]

❌ 有空格：skill001, skill002, skill003
   注入结果：["skill001", " skill002", " skill003"]（含前导空格！）

❌ 有空元素：skill001,,skill003
   注入结果：["skill001", "", "skill003"]（含空字符串）

✅ 防御性处理
Arrays.stream(lionService.formalSkillIds)
      .map(String::trim)     // 去除空格
      .filter(StringUtils::isNotBlank)  // 过滤空元素
      .collect(Collectors.toList())
```

## 3. 复杂类型（JSON）

### 3.1 HashMap 类型

```java
// LionService.java
@MdpConfig("ccs.metric.controller.condition")
public HashMap<Integer, Integer> metricControllerMap = Maps.newHashMap();

// Lion 控制台配置（JSON 格式）：
// key: ccs.metric.controller.condition
// value: {"1":7,"2":3,"3":10}

// 注入后：{1→7, 2→3, 3→10}
// key "1" → Integer 1，value "7" → Integer 7（框架自动转换）
```

### 3.2 HashMap 的使用

```java
// StaffPigeonService.java
public Map<String, List<String>> filterSkillStaffByMisCodes(List<String> misCodes, Date now) {
    // 从 Lion 获取各指标类型对应的天数
    Map<Integer, Integer> metricControllerMap = lionService.getMetricControllerMap();

    // 获取技能认证类型（code=1）对应的天数
    Integer days = Optional.ofNullable(
        metricControllerMap.get(MetricTypeEnum.SKILL_CERTIFICATION.getCode())
    ).orElse(0);   // Lion 未配置时默认0天

    Date expireDate = DateUtils.addDays(now, days);
    // 查询 days 天内到期的技能...
}
```

### 3.3 Lion 配置 Map 的常见格式

```
✅ 正确：标准 JSON 对象
{"1":7,"2":3,"3":10}

✅ 带空格（可接受）：
{ "1": 7, "2": 3 }

❌ key 用 Integer（JSON key 必须是字符串）：
{1:7, 2:3}  → JSON 解析失败

❌ 单引号（不是合法 JSON）：
{'1':7,'2':3}  → JSON 解析失败
```

## 4. 带默认值的多种写法

### 4.1 注解内默认值

```java
// 格式：@MdpConfig("key:defaultValue")
@MdpConfig("incentive.room.size:50")
private Integer roomSize;
// Lion 未配置 → 50
// Lion 配置了 100 → 100
```

### 4.2 字段初始值

```java
@MdpConfig("page.size")
private Integer pageSize = 200;
// Lion 未配置 → 200（字段初始值）
// Lion 配置了 500 → 500（Lion 值覆盖字段初始值）
```

### 4.3 两者同时存在

```java
// 优先级：Lion值 > 注解内默认值 > 字段初始值
@MdpConfig("page.size:300")
private Integer pageSize = 200;
// Lion 未配置 → 300（注解内默认值，不是字段初始值！）
// Lion 配置了 500 → 500
```

### 4.4 最佳实践建议

```java
// ✅ 推荐：注解内默认值，语义最清晰
@MdpConfig("page.size:200")
private Integer pageSize;

// 这样一看就知道：
// - 配置 key 是 page.size
// - 默认值是 200
// - 运行时可通过 Lion 动态修改

// ❌ 不推荐：两处默认值（容易混淆）
@MdpConfig("page.size:300")
private Integer pageSize = 200;   // 实际默认是 300，不是 200，容易误解
```

## 5. 类型转换规则

### 5.1 框架自动转换

MDP 框架在注入 Lion 配置时自动完成类型转换：

```
Lion 存储的值（都是字符串）    注解字段类型    实际注入的值
"100"                        Integer        100
"100"                        Long           100L
"3.14"                       Double         3.14
"true" / "false"             Boolean        true / false
"a,b,c"                      String[]       ["a", "b", "c"]
'{"k":"v"}'                  HashMap        {k→v}
```

### 5.2 类型转换失败的后果

```java
// Lion 配置值："abc"（非数字）
// 注解字段类型：Integer

@MdpConfig("page.size:200")
private Integer pageSize;

// Lion 配置了 "abc" → 类型转换失败
// 后果：字段保持上次成功拉取的值（或默认值）
// 框架打错误日志，但不会让应用崩溃
```

**防御写法**：

```java
// 避免直接使用可能 null 的字段
public int getPageSize() {
    return pageSize != null ? pageSize : 200;   // 有兜底
}
```
