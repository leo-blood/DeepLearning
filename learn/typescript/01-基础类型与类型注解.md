# 01 基础类型与类型注解

> 基于 `cs-fe-ess` 项目真实代码讲解，所有示例均来自或改编自生产代码。

---

## 1. 为什么需要类型注解

TypeScript 在 JavaScript 上增加了静态类型系统。没有类型时，一个函数的入参和返回值全靠文档（或者猜）：

```js
// JS：你不知道 date 是 string 还是 Date，也不知道返回什么
function transformDate(date, format) { ... }
```

加上类型注解后，IDE 实时提示，拼错立刻报红：

```ts
// TS：入参和返回值一目了然
function transformDate(date: Date | string, format: string): string { ... }
```

## 2. 原始类型

```ts
const name: string = '王文杰';
const age: number = 26;
const loading: boolean = false;
const nothing: null = null;
const undef: undefined = undefined;
```

> **项目中的例子** — `src/hooks/performance.ts`
>
> ```ts
> const loading = ref(true);          // ref<boolean>
> const salaryDate = ref<Date | string>(new Date());
> ```
>
> Vue 的 `ref` 会从初始值推断类型，但当初始值是 `new Date()` 而你后续还会赋字符串时，
> 就需要显式写 `ref<Date | string>`。

## 3. 联合类型 `A | B`

变量可以是多种类型之一：

```ts
let value: string | number;
value = '收益';  // ✅
value = 3200;    // ✅
value = true;    // ❌ 类型"boolean"不可赋值给"string | number"
```

**项目中的例子** — `src/hooks/performance.ts`：

```ts
const amount = computed(() => {
  const { totalSalary } = earnings.value || {};
  if (!totalSalary) return '本月无收益';
  if (totalSalary.value == null || isNaN(Number(totalSalary.value))) return totalSalary.value;
  return Number(totalSalary.value).toFixed(2);
});
```

`totalSalary.value` 可能是 `string | undefined`，所以先做 null 判断再 `Number()` 转换。

## 4. 数组类型

```ts
// 写法一：类型[]
const names: string[] = ['Alice', 'Bob'];

// 写法二：Array<类型>（泛型写法，后面章节详细讲）
const ids: Array<number> = [1, 2, 3];
```

**项目中的例子** — `src/hooks/myCourse.ts`：

```ts
const searchHistory = ref<string[]>([]);
```

`ref<string[]>` 表示这是一个响应式的字符串数组。

## 5. 对象类型（内联）

```ts
const user: { id: number; name: string } = { id: 1, name: '张三' };
```

内联对象类型适合一次性使用。复用场景应定义 `interface` 或 `type`（下一章讲）。

## 6. `any` vs `unknown`

| 类型      | 能随意访问属性？ | 类型安全？ |
|-----------|--------------|----------|
| `any`     | ✅            | ❌        |
| `unknown` | ❌（需收窄）  | ✅        |

```ts
// any：关闭类型检查，尽量避免
const data: any = fetchSomething();
data.foo.bar;  // 不报错，但运行时可能崩溃

// unknown：必须先判断类型才能使用
const res: unknown = fetchSomething();
if (typeof res === 'string') {
  console.log(res.toUpperCase());  // ✅ 安全使用
}
```

**项目中的例子** — `src/typings/index.d.ts`：

```ts
interface Window {
  KNB: any;  // 第三方 SDK，类型结构不确定，暂用 any
}
```

## 7. 类型推断

TypeScript 会从初始值 **自动推断** 类型，不需要每个变量都写注解：

```ts
const message = 'hello';   // 推断为 string
const count = 0;           // 推断为 number
const flag = true;         // 推断为 boolean
```

**最佳实践**：
- 有初始值时 → 让 TS 自动推断，不用写
- 没有初始值，或类型比推断结果更宽泛时 → 显式注解

---

## 8. 类型断言 `as`

告诉编译器"我比你更了解这个值的类型"：

```ts
const input = document.getElementById('search') as HTMLInputElement;
input.value = '关键词';  // 不加 as 时，getElementById 返回 HTMLElement | null，没有 .value
```

**项目中的例子** — `src/hooks/performance.ts`：

```ts
activityDetail.value = res.data as RewardActivityEventVo[];
```

API 返回 `unknown`，断言为具体类型后才能使用类型提示。

---

## 9. 非空断言 `!`

```ts
const el = document.getElementById('app')!;
// 等价于 as HTMLElement，告诉 TS 这不可能是 null
```

谨慎使用，如果运行时确实是 `null` 会抛错。

## 练习

基于项目代码，补全下面的类型注解：

```ts
// 题 1：补全参数类型和返回类型
function formatMonth(date: ___, format: ___): ___ {
  return dayjs(date).format(format);
}

// 题 2：给数组元素加类型
const tabList = [
  { code: 'IN_PROGRESS', label: '进行中' },
  { code: 'FINISHED',    label: '已完成' },
];
// 应该是什么类型？
```

<details>
<summary>参考答案</summary>

```ts
// 题 1
function formatMonth(date: Date | string, format: string): string {
  return dayjs(date).format(format);
}

// 题 2
const tabList: { code: string; label: string }[] = [
  { code: 'IN_PROGRESS', label: '进行中' },
  { code: 'FINISHED',    label: '已完成' },
];
```

</details>
