# 06 工具类型与高级类型操作

---

## 1. 内置工具类型速查

TypeScript 标准库内置了大量工具类型，项目中频繁使用：

| 工具类型 | 作用 | 例子 |
|---------|------|------|
| `Partial<T>` | 所有属性变可选 | `Partial<IUser>` |
| `Required<T>` | 所有属性变必填 | `Required<IConfig>` |
| `Readonly<T>` | 所有属性变只读 | `Readonly<IState>` |
| `Record<K, V>` | 键值对映射 | `Record<string, number>` |
| `Pick<T, K>` | 选取指定属性 | `Pick<IUser, 'id' \| 'name'>` |
| `Omit<T, K>` | 排除指定属性 | `Omit<IUser, 'password'>` |
| `Exclude<T, U>` | 从联合类型排除 | `Exclude<'a'\|'b'\|'c', 'a'>` → `'b'\|'c'` |
| `Extract<T, U>` | 从联合类型提取 | `Extract<string\|number, string>` → `string` |
| `NonNullable<T>` | 去掉 null/undefined | `NonNullable<string \| null>` → `string` |
| `ReturnType<T>` | 函数返回值类型 | `ReturnType<typeof fn>` |
| `Parameters<T>` | 函数参数类型元组 | `Parameters<typeof fn>` |

## 2. `Record<K, V>` — 项目高频使用

**项目中的直接使用** — `src/hooks/myCourse.ts`：

```ts
// myCourseList 的接口定义
myCourseList: Record<LearningStatus, MyLearningTaskVo[]>;

// 对应的实现
const myCourseList = reactive({
  [LearningStatus.IN_PROGRESS]:   [] as MyLearningTaskVo[],
  [LearningStatus.FINISHED]:      [] as MyLearningTaskVo[],
  [LearningStatus.NOT_FINISHED]:  [] as MyLearningTaskVo[],
});

// 使用时：
myCourseList[LearningStatus.IN_PROGRESS] = data?.processingLearningTaskList ?? [];
```

`Record<LearningStatus, MyLearningTaskVo[]>` 保证了：
- key 只能是 `LearningStatus` 枚举值（类型安全的枚举遍历）
- value 一定是 `MyLearningTaskVo[]`

## 3. `keyof` — 获取对象所有键的联合类型

```ts
interface IUser {
  id: number;
  name: string;
  email: string;
}

type UserKeys = keyof IUser;  // 'id' | 'name' | 'email'

// 实用场景：安全的属性访问
function getField<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user: IUser = { id: 1, name: '张三', email: 'zhang@example.com' };
getField(user, 'name');   // ✅ 返回 string
getField(user, 'avatar'); // ❌ 编译错误，'avatar' 不在 IUser 中
```

## 4. `typeof` — 从值获取类型

```ts
const config = {
  apiUrl: 'https://api.example.com',
  timeout: 5000,
};

type Config = typeof config;
// { apiUrl: string; timeout: number; }

// 常见用法：从常量对象提取类型
const STATUS_MAP = {
  active:   1,
  inactive: 0,
  deleted:  -1,
} as const;

type StatusKey   = keyof typeof STATUS_MAP;    // 'active' | 'inactive' | 'deleted'
type StatusValue = typeof STATUS_MAP[StatusKey]; // 1 | 0 | -1
```

## 5. `as const` — 字面量类型收窄

```ts
// 没有 as const：类型被推断为 string[]
const colors = ['red', 'green', 'blue'];
// colors: string[]

// 加了 as const：每个元素都是字面量类型，数组只读
const colors = ['red', 'green', 'blue'] as const;
// colors: readonly ['red', 'green', 'blue']

type Color = typeof colors[number];  // 'red' | 'green' | 'blue'
```

## 6. 映射类型（Mapped Types）

工具类型的底层实现原理：

```ts
// Partial 的实现
type MyPartial<T> = {
  [K in keyof T]?: T[K];
};

// Readonly 的实现
type MyReadonly<T> = {
  readonly [K in keyof T]: T[K];
};

// 自定义：给所有值加上 null
type Nullable<T> = {
  [K in keyof T]: T[K] | null;
};
```

## 7. 模板字面量类型

type EventName = 'click' | 'focus' | 'blur';

## 8. 索引访问类型 `T[K]`

```ts
interface ICourse {
  id: number;
  name: string;
  skillList: { subBusinessName: string; schemeType: string }[];
}

type SkillItem = ICourse['skillList'][number];
// { subBusinessName: string; schemeType: string }

type CourseName = ICourse['name'];
// string
```

**项目中的等价场景**：

```ts
// src/hooks/myCourse.ts
const businessFilter = ({ skillList }: MyLearningTaskVo): string[] =>
  skillList?.filter(Boolean).map(({ subBusinessName, schemeType }) =>
    `${subBusinessName}-${schemeType}`
  ) ?? [];
```

`skillList` 的元素类型（`MyLearningTaskVo['skillList'][number]`）就是解构出的 `{ subBusinessName, schemeType }` 所对应的类型。

## 9. 条件类型与 `infer`

`infer` 在条件类型中"捕获"类型：

```ts
// 提取 Promise 内部的类型
type Awaited<T> = T extends Promise<infer R> ? R : T;

type A = Awaited<Promise<string>>;  // string
type B = Awaited<number>;           // number（不是 Promise，直接返回 T）
```

这正是 TypeScript 5.x 内置 `Awaited<T>` 的实现原理。

**项目中的实际意义**：

```ts
async function fetchMyCourseList(): Promise<void> { ... }
type Result = Awaited<ReturnType<typeof fetchMyCourseList>>;
// void
```

## 10. 辨别联合类型（Discriminated Unions）

当联合类型的每个成员都有一个**共同的字面量字段**时，TS 可以精准收窄：

```ts
type Result<T> =
  | { status: 'success'; data: T }
  | { status: 'error'; message: string }
  | { status: 'loading' };

function handleResult<T>(result: Result<T>) {
  switch (result.status) {
    case 'success':
      console.log(result.data);     // 这里 result 是 { status: 'success'; data: T }
      break;
    case 'error':
      console.log(result.message);  // 这里 result 是 { status: 'error'; message: string }
      break;
  }
}
```

**项目中的相关模式** — `courseNameFilter` 的返回类型：

```ts
// 'default' | 'keyword' 就是辨别字段
{ type: 'default' | 'keyword'; content: string }[]

// 渲染层可以 switch type：
segments.forEach(({ type, content }) => {
  if (type === 'keyword') {
    // 渲染高亮样式
  } else {
    // 渲染普通文本
  }
});
```
