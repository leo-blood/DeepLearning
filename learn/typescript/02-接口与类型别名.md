# 02 接口与类型别名

> 本章是 TypeScript 最核心的内容之一——如何描述"对象的形状"。

---

## 1. `interface` 接口

`interface` 描述一个对象应该长什么样：

```ts
interface IUser {
  id: number;
  name: string;
  email?: string;  // ? 表示可选属性
}
```

**项目中的例子** — `src/hooks/performance.ts` 完整接口定义：

```ts
interface IPerformance {
  earnings: Ref<CloudStaffEarningsVO | undefined>;
  schemeList: Ref<SchemeConfigVO | undefined>;
  loading: Ref<boolean>;
  isMultiScheme: Ref<boolean>;
  detail: Ref<SchemeEarningsVO | undefined>;
  salaryDate: Ref<Date | string>;
  amount: Ref<string | undefined>;
  activityDetail: Ref<RewardActivityEventVo[] | undefined>;
  getEarnings: (date?: Date | string) => void;
  getSchemeList: (query: IQuery) => void;
  getDetailInfo: (query: IQuery | LocationQueryRaw) => void;
  getDate: () => Promise<void>;
  getActivityDetail: (startTime: number, endTime: number) => Promise<void>;
}
```

这个接口作为 **Hook 函数的返回值类型**，调用方拿到返回值后能得到完整的属性和方法提示。

## 2. `type` 类型别名

`type` 给类型起一个名字，可以是任意类型：

```ts
type Status = 'loading' | 'success' | 'error';       // 联合类型
type ID = string | number;                             // 联合类型
type Point = { x: number; y: number };                // 对象类型
type Callback = (value: string) => void;               // 函数类型
```

**项目中的例子** — 枚举状态映射：

```ts
// src/hooks/myCourse.ts
type LearningStatus = 'IN_PROGRESS' | 'FINISHED' | 'NOT_FINISHED';
```

## 3. `interface` vs `type` 对比

| 能力                          | interface | type |
|-------------------------------|:---------:|:----:|
| 描述对象结构                   | ✅        | ✅   |
| 描述联合类型 `A \| B`          | ❌        | ✅   |
| 描述交叉类型 `A & B`           | ❌        | ✅   |
| 描述元组类型 `[string, number]`| ❌        | ✅   |
| 声明合并（同名自动合并）        | ✅        | ❌   |
| 继承（`extends`）              | ✅        | ✅（用 &） |

**经验法则**：
- 描述对象/类的结构 → `interface`（支持继承和合并）
- 联合类型、工具类型、复杂组合 → `type`

## 4. 接口继承 `extends`

**项目中的完美例子** — `src/hooks/myCourse.ts`：

```ts
interface IMyCourse {
  TAB_LIST: { code: LearningStatus; label: string }[];
  activeTab: Ref<LearningStatus>;
  showProgress: ComputedRef<boolean>;
  searchKeyword: Ref<string>;
  myCourseList: Record<LearningStatus, MyLearningTaskVo[]>;
  loading: Ref<boolean>;
  goStudy: (course: MyLearningTaskVo) => void;
  fetchMyCourseList: () => Promise<void>;
  // ...
}

// Desktop 版本继承 IMyCourse，额外增加一个 courseActionFilter
interface IMyCourseDesktop extends IMyCourse {
  courseActionFilter: (course: MyLearningTaskVo) => string;
}

// Mobile 版本继承 IMyCourse，额外增加搜索相关功能
interface IMyCourseMobile extends IMyCourse {
  handleEnterSearchPage: () => Promise<void>;
  handleCancel: () => Promise<void>;
  handleSearch: () => Promise<void>;
  searchHistory: Ref<string[]>;
  isSearchPage: ComputedRef<boolean>;
  showSearchHistory: Ref<boolean>;
}
```

**继承的好处**：避免重复，同时保留扩展性。Desktop 和 Mobile 共享相同的基础接口，各自扩展特有功能。

## 5. 可选属性与只读属性

```ts
interface IConfig {
  apiUrl: string;           // 必填
  timeout?: number;         // 可选
  readonly version: string; // 只读，初始化后不能修改
}

const config: IConfig = {
  apiUrl: 'https://api.example.com',
  version: '1.0.0',
};

config.apiUrl = 'new-url';    // ✅
config.version = '2.0.0';    // ❌ 无法分配到 "version" ，因为它是只读属性
```

## 6. 索引签名

当对象的 key 不固定时：

```ts
interface IDict {
  [key: string]: string;
}

const translations: IDict = {
  hello: '你好',
  world: '世界',
};
```

**项目中的例子** — `src/hooks/myCourse.ts`：

```ts
// Record<LearningStatus, MyLearningTaskVo[]> 等价于索引签名
const myCourseList = reactive({
  [LearningStatus.IN_PROGRESS]: [] as MyLearningTaskVo[],
  [LearningStatus.FINISHED]:    [] as MyLearningTaskVo[],
  [LearningStatus.NOT_FINISHED]: [] as MyLearningTaskVo[],
});
```

`Record<K, V>` 是 TypeScript 内置工具类型，等价于 `{ [key in K]: V }`。

---

## 7. 接口描述函数

```ts
interface IGreeter {
  (name: string): string;
}

const greet: IGreeter = (name) => `你好，${name}`;
```

更常见的写法是在接口中描述方法：

```ts
interface IApi {
  fetchData: (id: number) => Promise<string>;
  postData: (payload: object) => Promise<void>;
}
```

---

## 8. 交叉类型 `A & B`

用 `&` 合并多个类型（`type` 专属）：

```ts
type HasName = { name: string };
type HasAge  = { age: number };
type Person  = HasName & HasAge;  // { name: string; age: number }
```

功能上类似 `interface extends`，但 `extends` 更易读，冲突时报错更清晰。

## 练习

根据项目的 `IQuery` 接口，回答以下问题：

```ts
// src/hooks/performance.ts
interface IQuery {
  staffType?: string;
  workDeliverTeamId?: string;
  workSchemeId?: string;
  workSchemeTypeId?: string;
  workSubId?: string;
  workSubBizId?: string;
  billMonth?: string;
}
```

1. 所有字段都是可选的（`?`），这意味着什么？调用 `getSchemeList({})` 会报错吗？
2. 如何扩展 `IQuery`，增加一个必填的 `pageSize: number` 字段？

<details>
<summary>参考答案</summary>

1. 所有字段可选意味着传空对象 `{}` 合法，适合"按需传参"的 API 查询场景。`getSchemeList({})` 不会报错。

2. 有两种方式：

```ts
// 方式一：接口继承
interface IQueryWithPage extends IQuery {
  pageSize: number;
}

// 方式二：交叉类型
type IQueryWithPage = IQuery & { pageSize: number };
```

</details>
