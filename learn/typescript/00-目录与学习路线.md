## 快速参考：项目中的 TS 核心模式

### Hook 接口继承模式

```ts
// 基础接口
interface IMyCourse { ... }

// 平台扩展
interface IMyCourseDesktop extends IMyCourse {
  courseActionFilter: (course: MyLearningTaskVo) => string;
}

interface IMyCourseMobile extends IMyCourse {
  searchHistory: Ref<string[]>;
  handleSearch: () => Promise<void>;
  // ...
}
```

### 响应式类型模式

```ts
// ref → Ref<T>
const loading = ref(false);  // Ref<boolean>

// computed → ComputedRef<T>（只读）
const showProgress = computed(() => ...);  // ComputedRef<boolean>

// reactive → 直接推断
const state = reactive({ count: 0 });  // { count: number }
```

### API 调用类型模式

```ts
// 请求时
const { data } = await postStudentCenterGetMyLearningTask(
  searchKeyword.value ? { taskName: searchKeyword.value } : {}
);
// data 类型由 API 函数的泛型参数决定

// 接收时处理可能的 undefined
myCourseList[LearningStatus.IN_PROGRESS] = data?.processingLearningTaskList ?? [];
```

### 全局类型声明模式

```ts
// src/typings/index.d.ts — 全局注入
declare const envConfig: { apiOrigin: { user: string; ... } };
interface Window { KNB: any; }

// src/typings/api.d.ts — 全局 API 类型
// 无需 import，直接在任何文件中使用 MyLearningTaskVo
```
