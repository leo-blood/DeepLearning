## 1. 单文件组件（SFC）结构

Vue 的单文件组件（`.vue` 文件）包含三个块：

```vue
<template>
  <!-- HTML 结构 -->
</template>

<script setup lang="ts">
// 组件逻辑（Composition API）
</script>

<style scoped>
/* 样式（scoped 只作用于本组件） */
</style>
```

项目中存在两种写法，对应不同历史时期：

```vue
<!-- 写法一：script setup（Vue 3.2+，推荐） -->
<script setup lang="ts">
import { ref } from 'vue'
const count = ref(0)
</script>

<!-- 写法二：Options API + vue-class-component（项目遗留部分） -->
<script lang="ts">
import { Options, Vue } from 'vue-class-component';
@Options({ components: { ... } })
export default class Home extends Vue { ... }
</script>
```

## 2. 模板语法

### 2.1 文本插值 `{{ }}`

```vue
<template>
  <div>{{ message }}</div>
  <div>{{ count * 2 }}</div>
  <div>{{ isActive ? '激活' : '未激活' }}</div>
</template>
```

### 2.2 指令 `v-*`

**项目中的真实模板** — `src/views/app/pages/SignContract/index.vue`：

```vue
<template>
  <!-- v-if：条件渲染 -->
  <div
    v-if="showToast"
    :class="[
      'toast-wrap',
      {
        loading: toastStatus === toastStatuMap.loading,
        success: toastStatus === toastStatuMap.success,
        fail:    toastStatus === toastStatuMap.fail,
      },
    ]"
  >
    <!-- v-bind 简写 :   绑定动态属性 -->
    <img class="img" :src="toastModel[toastStatus].url" />
    <div class="text">{{ toastModel[toastStatus].text }}</div>
  </div>

  <!-- v-on 简写 @   绑定事件 -->
  <div :class="{ button: true, disabled: buttonDisabled }" @click="handleSign">
    {{ buttonText }}
  </div>
</template>
```

### 2.3 常用指令速查

| 指令 | 简写 | 作用 | 示例 |
|------|------|------|------|
| `v-bind` | `:` | 绑定属性/prop | `:src="url"` |
| `v-on` | `@` | 绑定事件 | `@click="fn"` |
| `v-if` | — | 条件渲染（真实 DOM 销毁创建） | `v-if="show"` |
| `v-show` | — | 条件显隐（CSS display 切换） | `v-show="show"` |
| `v-for` | — | 列表渲染 | `v-for="item in list"` |
| `v-model` | — | 双向绑定 | `v-model="value"` |
| `v-else-if` | — | 配合 v-if 使用 | `v-else-if="cond"` |
| `v-else` | — | 配合 v-if 使用 | `v-else` |

### 2.4 v-for 列表渲染

```vue
<template>
  <!-- 必须加 :key，帮助 Vue 追踪节点身份 -->
  <div v-for="item in courseList" :key="item.id" class="course-item">
    {{ item.name }}
  </div>

  <!-- 带索引 -->
  <div v-for="(item, index) in list" :key="index">
    {{ index + 1 }}. {{ item.name }}
  </div>
</template>
```

## 3. `<script setup>` — 现代写法

`<script setup>` 是 Vue 3.2 推出的语法糖，所有顶层变量/函数**自动暴露给模板**：

```vue
<script setup lang="ts">
import { ref, computed, onMounted } from 'vue'

// 顶层声明自动暴露给 template，无需 return
const count = ref(0)
const double = computed(() => count.value * 2)

function increment() {
  count.value++
}

onMounted(() => {
  console.log('组件已挂载')
})
</script>

<template>
  <button @click="increment">{{ count }} × 2 = {{ double }}</button>
</template>
```

**对比 Options API（项目遗留写法）**：

```vue
<script lang="ts">
export default {
  data() {
    return { count: 0 }
  },
  computed: {
    double() { return this.count * 2 }
  },
  methods: {
    increment() { this.count++ }
  },
  mounted() {
    console.log('组件已挂载')
  }
}
</script>
```

## 5. Emits（子传父）

```vue
<!-- 子组件 -->
<script setup lang="ts">
const emit = defineEmits<{
  (e: 'change', value: string): void
  (e: 'close'): void
}>()

function handleInput(value: string) {
  emit('change', value)  // 触发事件，携带参数
}

function handleClose() {
  emit('close')
}
</script>
```

```vue
<!-- 父组件 -->
<SearchInput
  @change="handleChange"
  @close="handleClose"
/>
```

## 7. 模板引用 `ref` — 访问 DOM 元素

```vue

  <input ref="inputRef" type="text" />
```

## 8. 计算属性 vs 方法

```vue
<script setup lang="ts">
import { ref, computed } from 'vue'

const firstName = ref('文')
const lastName  = ref('杰')

// ✅ computed：有缓存，依赖不变不重新计算
const fullName = computed(() => `${lastName.value}${firstName.value}`)

// ❌ 方法：每次渲染都会调用，无缓存
function getFullName() {
  return `${lastName.value}${firstName.value}`
}
</script>
```

## 9. 路由：useRouter / useRoute

```ts
import { useRouter, useRoute } from 'vue-router'

// useRouter：操作路由（跳转）
const router = useRouter()
router.push('/square')             // 跳转
router.replace('/learning/course') // 替换（不记录历史）

const route = useRoute()
```

## 练习

分析 `src/views/app/pages/SignContract/index.vue` 中的倒计时逻辑：

```ts
const countdown = ref(30)
const countdownTimer = ref<ReturnType<typeof setInterval> | null>(null)

const startCountdown = () => {
  buttonText.value = `已阅读协议（${countdown.value}s）`
  countdownTimer.value = setInterval(() => {
    countdown.value -= 1
    if (countdown.value > 0) {
      buttonText.value = `已阅读协议（${countdown.value}s）`
    } else {
      clearInterval(countdownTimer.value!)
      countdownTimer.value = null
      buttonDisabled.value = false
      buttonText.value = '我已阅读并同意签署'
    }
  }, 1000)
}
```

1. `ReturnType<typeof setInterval>` 是什么类型？为什么要写成这样而不是直接写 `number`？
2. `clearInterval(countdownTimer.value!)` 末尾的 `!` 是什么意思？为什么这里可以安全使用？
3. 如果组件被销毁时倒计时还在进行，会有什么问题？如何用生命周期钩子修复？

<details>
<summary>参考答案</summary>

1. `setInterval` 在浏览器中返回 `number`，但在 Node.js 中返回 `NodeJS.Timeout`。`ReturnType<typeof setInterval>` 让 TS 自动推断出当前运行环境的正确类型，比硬写 `number` 更跨平台。

2. `!` 是非空断言，告诉 TS "这里 `countdownTimer.value` 肯定不是 null"。在 `else` 分支进入时，`countdownTimer.value` 一定是由 `setInterval` 赋值过的，所以断言是安全的。

3. 组件销毁后 `countdownTimer` 仍然在运行，会尝试修改已销毁组件的响应式数据，可能导致内存泄漏或警告。修复：

```ts
import { onUnmounted } from 'vue'

onUnmounted(() => {
  if (countdownTimer.value) {
    clearInterval(countdownTimer.value)
  }
})
```

</details>
