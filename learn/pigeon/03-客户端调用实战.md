# Pigeon 客户端调用实战

> 本文聚焦：如何从本项目（ccsadmin）调用其他系统的 Pigeon 服务。

---

## 1. @MdpPigeonClient 两种注入方式

### 1.1 方式一：字段注入（最常用）

直接在 Service 类的字段上加注解：

```java
// StaffPigeonService.java
@Service
@Slf4j
public class StaffPigeonService {

    // 注入 MDM 系统的员工信息服务（10秒超时）
    @MdpPigeonClient(timeout = 10000)
    IStaffPigeonService iStaffPigeonService;

    // 注入 csc-cratos 的员工统计服务（10秒超时）
    @MdpPigeonClient(timeout = 10000)
    IStaffRemoteService iStaffRemoteService;

    public Optional<StaffDTO> getStaffByStaffMis(String staffMis) {
        if (staffMis == null) return Optional.empty();
        try {
            BaseResultDTO<StaffDTO> res =
                iStaffRemoteService.queryByLoginNameFromCache(staffMis);
            return Optional.ofNullable(res.isSuccess() ? res.getData() : null);
        } catch (Exception e) {
            log.error("查询员工失败, staffMis:{}", staffMis, e);
            return Optional.empty();
        }
    }
}
```

### 1.2 方式二：@Configuration 类注入（特殊配置时）

```java
// PigeonClientConfig.java
@Configuration
public class PigeonClientConfig {

    // 技能服务超时较短（3秒），单独配置
    @MdpPigeonClient(timeout = 3000L)
    private PacificSkillRemoteService pacificSkillRemoteService;
}
```

**何时用 Configuration 注入**：
- 需要覆盖默认超时，且该接口在多处使用
- 需要特殊的负载均衡或路由配置
- 引入的接口没有被 `@Service` 等扫描到

### 1.3 @MdpPigeonClient 参数

```java
@MdpPigeonClient(
    timeout = 5000L,      // 超时时间（毫秒），默认 3000
    callType = ...,       // 调用方式（同步/异步），默认同步
    cluster = ...         // 指定集群（一般不需要设置）
)
IStaffPigeonService iStaffPigeonService;
```

## 3. 超时策略

### 3.1 超时时间选择原则

```java
// 快速单点查询：1~3秒
@MdpPigeonClient(timeout = 3000L)
PacificSkillRemoteService pacificSkillRemoteService;

// 普通查询：3~5秒
@MdpPigeonClient(timeout = 5000)
ITrainingRemoteService trainingRemoteService;

// 批量查询（数据量大）：8~10秒
@MdpPigeonClient(timeout = 10000)
IStaffPigeonService iStaffPigeonService;

// 写操作（需确保完成）：5~8秒
@MdpPigeonClient(timeout = 8000)
ICloudOfficeStaffLabelService cloudOfficeService;
```

### 3.2 超时的影响

```
超时太短：
  Provider 还在正常处理，Consumer 已经超时报错
  → 误判服务不可用，错误日志增多

超时太长：
  Provider 出问题，Consumer 线程被长时间阻塞
  → 线程池耗尽，整个服务不可用（雪崩）

原则：超时 = Provider 正常 P99 响应时间 × 3~5
     同时不超过 HTTP 接口的超时时间
```

## 4. 防御性编程三板斧

### 4.1 入参校验

```java
// 避免发起无意义的 RPC 调用
public Map<String, StaffDTO> getStaffByStaffMis(List<String> staffMisList) {
    // 校验：空列表不发请求
    if (CollectionUtils.isEmpty(staffMisList)) {
        return Maps.newHashMap();
    }
    // ... 发起 RPC
}

public Optional<StaffDTO> getStaffByStaffId(Long staffId) {
    // 校验：非法 ID 不发请求
    if (staffId == null || staffId <= 0) {
        log.warn("查询员工信息失败，staffId无效:{}", staffId);
        return Optional.empty();
    }
    // ... 发起 RPC
}
```

### 4.2 结果校验

```java
// RPC 调用成功 ≠ 业务成功，必须检查业务结果
public Map<String, StaffDTO> getStaffByStaffMis(List<String> staffMisList) {
    BaseResultDTO<List<StaffDTO>> resultDTO =
        iStaffRemoteService.listStaffByLoginNames(staffMisList, null);

    // ✅ 必须判断业务是否成功
    if (resultDTO.isSuccess()) {
        return resultDTO.getData().stream()
            .collect(Collectors.toMap(StaffDTO::getLoginName, Function.identity(), (k1, k2) -> k1));
    } else {
        // 业务失败：记录日志，返回空
        log.warn("查询员工列表失败, errorMsg:{}", resultDTO.getMessage());
        return Maps.newHashMap();
    }
}
```

### 4.3 异常捕获降级

```java
// 所有 RPC 调用必须在 try-catch 中，异常时降级
public Optional<StaffDTO> getStaffByStaffMis(String staffMis) {
    if (staffMis == null) return Optional.empty();
    try {
        BaseResultDTO<StaffDTO> res =
            iStaffRemoteService.queryByLoginNameFromCache(staffMis);
        return Optional.ofNullable(res.isSuccess() ? res.getData() : null);
    } catch (Exception e) {
        // 网络超时、服务不可用等情况 → 降级返回空，不抛异常
        log.error("{}, 查询员工失败, staffMis:{}",
            CommonConstant.LOG_PREFIX_CLOUD_CUSTOMER_SERVICE_PLATFORM, staffMis, e);
        return Optional.empty();
    }
}
```

## 5. 批量调用与分页

### 5.1 为什么需要分批

```java
// ❌ 一次传 5000 个 MIS → Provider 端处理超时
iStaffPigeonService.batchGetStaffInfoByMisCodes(all5000MisList);

// ✅ 每次传 20 个，循环调用
List<List<String>> batches = Lists.partition(new ArrayList<>(staffMisList), 20);
for (List<String> batch : batches) {
    iStaffPigeonService.batchGetStaffInfoByMisCodes(batch);
}
```

### 5.2 本项目的批量实现

```java
// StaffPigeonService.java
public Map<String, StaffRespDto> getStaffInfoByMisCode(Set<String> staffMisList) {
    List<StaffRespDto> allResults = new ArrayList<>();

    // 分批：每次 20 条
    List<List<String>> partition = Lists.partition(new ArrayList<>(staffMisList), 20);

    for (List<String> batch : partition) {
        StaffRequestDTO request = new StaffRequestDTO();
        request.setBizcode(BIZ_CODE);
        request.setAppkey(APP_KEY);
        request.setScope(SCOPE);
        request.setMisCodes(batch);

        try {
            BizResoponseDTO<List<StaffRespDto>> result =
                iStaffPigeonService.batchGetStaffInfoByMisCodes(request);
            if (result != null && result.isSuccess() && result.getData() != null) {
                allResults.addAll(result.getData());
            }
        } catch (Exception e) {
            log.error("批量查询员工失败, batch:{}", batch, e);
            // 单批失败，继续下一批（不中断整体）
        }
    }

    // 构建 MIS → StaffRespDto 的映射
    return buildMisToStaffMap(allResults, staffMisList);
}
```

### 5.3 大列表分批（10000条）

```java
// mapStaffDTOByStaffMis：最多一次传 10000 条
public Map<String, StaffDTO> mapStaffDTOByStaffMis(Set<String> staffMisList) {
    List<StaffDTO> allStaff = new ArrayList<>();

    // 每批 10000 条
    List<List<String>> batches = Lists.partition(new ArrayList<>(staffMisList), 10000);

    for (List<String> batch : batches) {
        try {
            BaseResultDTO<List<StaffDTO>> result =
                iStaffRemoteService.listStaffByLoginNames(batch, null);
            if (result.isSuccess()) {
                allStaff.addAll(result.getData());
            }
        } catch (Exception e) {
            log.error("批量查询员工详情失败（已忽略）", e);
        }
    }

    return allStaff.stream()
        .collect(Collectors.toMap(StaffDTO::getLoginName, Function.identity(), (k1, k2) -> k1));
}
```

## 6. 常见调用问题排查

### 6.1 ServiceUnavailableException

```
异常信息：com.dianping.pigeon.remoting.invoker.exception.ServiceUnavailableException

可能原因：
  1. Provider 未启动（本地测试时 MDM 没启动）
  2. Provider 的 AppKey 配置错误，注册到了错误的命名空间
  3. MNS 注册中心不可用（极罕见）

排查步骤：
  1. 检查 Provider 是否正常启动（查看 Provider 的日志）
  2. 检查 Provider 的 @MdpPigeonServer 注解是否加了
  3. 本地测试时，检查是否配置了正确的测试环境
```

### 6.2 PigeonTimeoutException

```
异常信息：com.dianping.pigeon.remoting.invoker.exception.PigeonTimeoutException

可能原因：
  1. Provider 处理慢（DB 慢查询、GC 停顿）
  2. 超时时间设置过短
  3. 网络问题

解决：
  短期：增大 @MdpPigeonClient(timeout = xxx)
  根本：找 Provider 团队优化接口性能
```

### 6.3 ClassNotFoundException（序列化版本不一致）

```
异常信息：Hessian deserialization error: ClassNotFoundException

可能原因：
  Consumer 和 Provider 使用了不同版本的 ccsadmin-api jar

解决：
  统一 pom.xml 中 <api.version> 的版本号
  与 Provider 团队确认应使用哪个版本
```
