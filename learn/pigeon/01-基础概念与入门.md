# Pigeon 基础概念与入门

---

## 1. 本地调用 vs 远程调用

### 1.1 本地方法调用

单体应用里，调用一个方法是这样的：

```java
// 直接调用，编译器知道地址，JVM 直接跳转
StaffDTO staff = staffService.queryByMis("zhangsan");
```

整个过程在一个 JVM 进程内完成，没有网络，耗时微秒级。

### 1.2 远程调用的挑战

微服务拆分后，`staffService` 可能部署在另一台机器：

```
你的机器（ccsadmin）            MDM 机器
    │                               │
    │  需要员工信息                  │
    │  但 staffService 在那边         │
    │                               │
    ?  怎么调用 ?
```

直接调用需要解决：
1. **序列化**：Java 对象 → 字节流（对方机器才能传输）
2. **网络传输**：TCP 建连、发包、接包
3. **反序列化**：字节流 → Java 对象
4. **服务发现**：对方机器的 IP 和端口是什么？
5. **超时处理**：网络不通或对方慢怎么办？
6. **负载均衡**：对方有10台机器，调哪台？

### 1.3 RPC 框架的价值

RPC（Remote Procedure Call）框架把上述所有复杂性封装起来，让你像调本地方法一样调用远端服务：

```java
// 有了 RPC 框架，这行代码实际上跨机器调用了
// 但写法和本地调用一模一样
StaffDTO staff = iStaffRemoteService.queryByLoginNameFromCache("zhangsan");
```

## 2. Pigeon 在美团的位置

### 2.1 Pigeon 是什么

Pigeon 是美团自研的 RPC 框架，2012年上线，目前承载美团内部绝大多数服务间调用。主要特性：

- 基于 **Netty** 的异步非阻塞网络通信
- 默认使用 **Hessian** 序列化协议
- 注册中心：美团内部 **MNS**（服务名字服务）
- 与 MDP 深度集成，两个注解搞定服务注册与发现

### 2.2 与 HTTP/REST 的对比

| 对比项 | Pigeon（RPC） | HTTP REST |
|--------|--------------|-----------|
| 协议 | 自定义二进制协议 | HTTP |
| 序列化 | Hessian（紧凑） | JSON（可读） |
| 性能 | 高（低开销） | 中（HTTP头部开销大） |
| 调用方式 | 接口方法调用 | URL + HTTP Method |
| 服务发现 | 自动（注册中心） | 手动（配置 URL） |
| 适用场景 | 内部服务间调用 | 对外 API、跨语言 |
| 错误处理 | 异常/返回码 | HTTP 状态码 |

**结论**：内部服务间高频调用用 Pigeon，对外暴露给前端/第三方用 HTTP。

### 2.3 本项目中 Pigeon 的规模

```
对外暴露（本项目作为 Server）：6 个服务
  ┌─────────────────────────────────────────────────────┐
  │ IncentiveServiceImpl         激励任务查询             │
  │ CcsRankRewardPigeonServiceImpl  排名奖励规则          │
  │ CcsOrgPigeonServiceImpl      组织架构数据             │
  │ RewardActivityRemoteService  奖励活动数据             │
  │ CcsTaskRewardServicePigeonImpl 任务奖惩规则           │
  │ RecruitmentTaskPigeonServiceImpl 招募任务信息         │
  └─────────────────────────────────────────────────────┘

调用外部（本项目作为 Client）：10+ 个服务
  ┌─────────────────────────────────────────────────────┐
  │ MDM 系统       员工信息、技能标签                      │
  │ 排班系统       班次信息、权限校验                      │
  │ 培训系统       课程内容、培训进度                      │
```

## 3. Pigeon 核心概念

### 3.1 Provider（服务提供方）

提供具体功能实现的应用，等待被调用。

```
本项目（ccsadmin）作为 Provider：
  实现了 IncentiveService 接口
  → 当 csc-cratos 系统调用 checkTrainingPacakge() 时，由本项目执行
```

### 3.2 Consumer（服务消费方）

调用其他服务的应用，发起远程调用。

```
本项目（ccsadmin）作为 Consumer：
  注入了 IStaffPigeonService 代理
  → 调用 batchGetStaffInfoByMisCodes() 时，实际由 MDM 系统执行
```

### 3.3 注册中心（MNS）

类比于"电话黄页"，记录每个服务的 IP 地址：

```
Provider 启动时：向 MNS 注册 → "我是 IncentiveService，地址是 10.0.1.23:5001"

Consumer 调用时：查 MNS → "IncentiveService 在哪里？"
               → MNS 返回地址列表 → Consumer 选一台发起调用
```

### 3.4 序列化

Java 对象不能直接在网络上传输，需要转换为字节序列：

```
Consumer 端：
  Java 对象（StaffRequestDTO）──[Hessian序列化]──▶ 字节数组 ──▶ 网络传输

Provider 端：
  字节数组 ──[Hessian反序列化]──▶ Java 对象（StaffRequestDTO）──▶ 方法调用
```

**因此接口参数和返回值必须可序列化**（实现 Serializable 或符合 Hessian 规范）。

### 3.5 服务接口（Interface）

Pigeon 调用是基于 Java 接口的：

```
接口定义（ccsadmin-api 模块，打成 jar 供他人引用）：
  public interface IncentiveService {
      ResultVO<Boolean> checkTrainingPacakge(Long packageId);
  }

实现类（ccsadmin-starter 模块，本项目内部）：
  @MdpPigeonServer
  public class IncentiveServiceImpl implements IncentiveService { ... }

Consumer 引用（其他系统，引用 ccsadmin-api jar）：
  @MdpPigeonClient
  IncentiveService incentiveService;  // 只需接口，不需要实现类
```

## 4. 调用链路全貌

```
Consumer 代码调用
    │
    ▼
Pigeon Client Proxy（动态代理，拦截方法调用）
    │
    ├── 序列化请求参数（Hessian）
    ├── 向 MNS 查询 Provider 地址（有缓存，不每次查）
    ├── 负载均衡选一台 Provider
    │
    ▼
网络传输（Netty TCP 长连接）
    │
    ▼
Pigeon Server（Provider 端）
    │
    ├── 反序列化请求参数
    ├── 从线程池取一个线程
    ├── 调用实际业务方法
    ├── 序列化返回值
    │
    ▼
网络传输（返回）
    │
    ▼
Pigeon Client 接收响应
    │
    ├── 反序列化返回值
    ├── 返回给 Consumer 代码
    │
    ▼
Consumer 代码拿到结果，与本地调用无区别
```

## 5. 第一个 Pigeon 程序（概念验证）

### 5.1 定义接口（api 模块）

```java
// ccsadmin-api/src/main/java/.../api/service/IncentiveService.java
// 这个接口会打成 jar，供调用方引用

public interface IncentiveService {
    /**
     * 检查培训内容包是否可删除
     * @param packageId 内容包ID
     * @return true=可删除（无关联任务），false=不可删除（有关联任务）
     */
    ResultVO<Boolean> checkTrainingPacakge(Long packageId);
}
```

### 5.2 实现接口（starter 模块）

```java
// ccsadmin-starter/.../pigeon/server/IncentiveServiceImpl.java

@MdpPigeonServer    // 一个注解完成注册
@Slf4j
public class IncentiveServiceImpl implements IncentiveService {

    @Autowired
    private IncentiveMapperService incentiveMapperService;

    @Override
    public ResultVO<Boolean> checkTrainingPacakge(Long packageId) {
        try {
            List<IncentiveTask> tasks = incentiveMapperService
                .queryUnFinishTaskByContent(CONTENT_TASK.getCode(), packageId);
            boolean canDelete = CollectionUtils.isEmpty(tasks);
            return ResultVO.newSuccessObject(canDelete);
        } catch (Exception e) {
            log.error("checkTrainingPacakge error, packageId:{}", packageId, e);
            return ResultVO.newServerErrorObject();  // 不能抛异常！
        }
    }
}
```

### 5.3 调用接口（其他系统）

```java
// 其他系统引入 ccsadmin-api jar 后

@Service
public class SomeOtherService {

    @MdpPigeonClient(timeout = 3000)
    IncentiveService incentiveService;   // 注入远程服务代理

    public void deletePackage(Long packageId) {
        ResultVO<Boolean> result = incentiveService.checkTrainingPacakge(packageId);
        if (result.isSuccess() && result.getData()) {
            // 可以删除
        } else {
            throw new BusiException("内容包已被激励任务引用，无法删除");
        }
    }
}
```

## 6. 关键约束与注意事项

| 约束 | 说明 |
|------|------|
| 接口参数必须可序列化 | 不能传 InputStream、Connection、Lambda 等 |
| 方法不能抛 checked exception | 只允许返回包含错误信息的对象 |
| API jar 需要版本管理 | Consumer 和 Provider 必须用相同版本的接口定义 |
| 不能传递 ThreadLocal 数据 | 登录用户等上下文信息需显式传参 |
| 超时时间必须设置 | 默认超时可能不适合你的场景 |
