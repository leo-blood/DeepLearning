## 1. 定时任务的挑战

### 1.1 单机 @Scheduled 的致命缺陷

Spring 的 `@Scheduled` 在每台机器上独立运行：

```java
@Scheduled(cron = "0 0 1 * * ?")  // 每天凌晨1点
public void calcStaffLevel() {
    // 计算员工等级
}
```

**生产环境部署了3台机器**：

```
凌晨1点整：
  机器A  ──▶ calcStaffLevel() 开始执行
  机器B  ──▶ calcStaffLevel() 开始执行   ← 重复！
  机器C  ──▶ calcStaffLevel() 开始执行   ← 重复！
```

后果：员工等级被计算3次，数据互相覆盖，最终结果不可预期。

### 1.2 单机加分布式锁的问题

很多团队用这种方式：

```java
@Scheduled(cron = "0 0 1 * * ?")
public void calcStaffLevel() {
    boolean locked = redis.setnx("calcStaffLevel_lock", 1, 3600);
    if (!locked) return;  // 抢不到锁就退出
    try {
        doCalc();
    } finally {
        redis.del("calcStaffLevel_lock");
    }
}
```

**还有问题**：
- 修改执行时间要重新部署
- 无法手动触发（临时补跑数据）
- 没有执行历史记录，排查困难
- 无法传参（无法指定执行哪天的数据）
- Redis 故障导致所有机器抢锁失败，任务不执行

### 1.3 分布式调度系统的解法

```
Crane 控制台（统一管理所有任务）
    │
    ├── cron 时间到达 → 从注册机器中选1台 → 发指令执行
    ├── 执行历史完整记录（时间/机器/耗时/结果）
    ├── 支持手动触发（补跑任何日期的数据）
    ├── 支持传参（日期、批次等）
    ├── 支持修改 cron（不用重启服务）
    └── 支持关闭/暂停任务
```

## 2. Crane 架构原理

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────┐
│                   Crane 控制台                       │
│  配置 cron │ 手动触发 │ 查看历史 │ 管理机器           │
└─────────────┬───────────────────────────────────────┘
              │ HTTP/RPC 调度指令
              ▼
┌─────────────────────────────────────────────────────┐
│              Crane Agent（每台应用机器上）             │
│                                                     │
│  接收调度指令 → 反射调用 @Crane 方法 → 上报执行结果    │
│                                                     │
│  机器A  @Crane("ccsStaffLevelJob") → calcStaffLevel │
│  机器B  @Crane("ccsStaffLevelJob") → calcStaffLevel │ ← 只有被选中的机器执行
│  机器C  @Crane("ccsStaffLevelJob") → calcStaffLevel │
└─────────────────────────────────────────────────────┘
```

### 2.2 任务注册流程

```
应用启动
  1. Crane SDK 扫描所有 @Crane 注解
  2. 收集 {taskName, methodRef, appKey, ip, port}
  3. 上报给 Crane 控制台注册

Crane 控制台
  4. 记录该任务的机器列表
  5. 按 cron 定时触发，选择一台机器执行
```

## 3. @Crane 注解语法

### 3.1 基础语法

```java
import com.cip.crane.client.spring.annotation.Crane;

@Service
public class CcsStaffLevelJob {

    // 注解参数就是控制台上的任务名
    @Crane("ccsStaffLevelJob")
    public void calcStaffLevel(String param) {
        // param 是控制台传来的参数，可以是日期、配置等
        log.info("员工等级计算开始, param:{}", param);
    }
}
```

### 3.2 参数说明

| 元素 | 说明 | 示例 |
|------|------|------|
| 注解值（任务名） | 控制台上的唯一标识，用于配置和触发 | `"ccsStaffLevelJob"` |
| 方法参数 | 固定为 `String param`，接收控制台传来的参数 | `"2024-01-15"` |
| 方法返回值 | 通常为 void，框架通过异常判断执行结果 | `void` |
| 所在类 | 必须是 Spring Bean（`@Service`/`@Component`） | `@Service` |

### 3.3 一个类中多个 Crane 任务

```java
// IncentiveTaskJob.java —— 激励任务相关 Job 都在这里
@Service
@Slf4j
public class IncentiveTaskJob {

    @Crane("incentiveStatusUpdate")      // 任务1：状态流转
    public void updateTaskStatus(String dateStr) {
        log.info("激励任务状态流转, dateStr:{}", dateStr);
        // ...
    }

    @Crane("doIncentiveTask")            // 任务2：执行圈人/推送
    public void doIncentiveTask(String dateStr) {
        log.info("激励任务执行, dateStr:{}", dateStr);
        // ...
    }
}
```

两个方法在控制台上是两个独立的任务，可以配置不同的 cron 和参数。

## 4. Crane 控制台核心功能

### 4.1 任务列表

控制台展示所有注册的任务：

```
任务名                    cron          上次执行时间      结果    耗时
ccsStaffLevelJob         0 0 1 * * ?   2024-01-15 01:00  成功   4m32s
taskRewardPenaltyJob     0 0 23 * * ?  2024-01-14 23:00  成功   1m15s
incentiveStatusUpdate    0 0 2 * * ?   2024-01-15 02:00  成功   22s
RuleReminderJob          0 0 1 * * ?   2024-01-15 01:00  成功   8s
```

### 4.2 手动触发

```
操作场景：
  1. 测试：开发完成后，手动触发验证逻辑
  2. 补跑：昨天任务失败，手动补跑
  3. 临时需求：运营要求立即刷新员工等级

操作步骤：
  控制台 → 找到任务 → 点击"执行" → 填写参数（如日期） → 选择机器 → 确认
```

### 4.3 修改 cron（无需重启）

```
正常：0 0 1 * * ?（每天凌晨1点）
改为：0 30 2 * * ?（每天凌晨2点半）

→ 直接在控制台修改 → 秒级生效 → 不用重新部署应用
```

### 4.4 执行历史

```
任务名: ccsStaffLevelJob
执行时间        机器IP         参数        耗时    结果
2024-01-15 01:00  10.0.1.23    (无)       4m32s   成功
2024-01-14 01:00  10.0.1.24    (无)       4m18s   成功
2024-01-13 01:00  10.0.1.23    (无)       失败    异常信息: OOM...
```

可以看到：哪台机器执行了、执行了多久、成不成功、失败时的异常信息。

## 6. Crane vs 其他定时方案对比

| 方案 | 多实例重复执行 | 手动触发 | 执行历史 | 动态修改 cron | 传参 |
|------|-------------|---------|---------|-------------|------|
| `@Scheduled` | ❌ 重复 | ❌ | ❌ | ❌（需重启）| ❌ |
| `@Scheduled` + Redis锁 | ✅ | ❌ | ❌ | ❌ | ❌ |
| **Crane** | ✅ | ✅ | ✅ | ✅ | ✅ |
| Quartz 集群 | ✅ | 需自己开发 | 需自己开发 | 需重启 | ✅ |
