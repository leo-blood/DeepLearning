# Crane 任务开发实战

> 本文聚焦：如何编写一个完整、可靠的 Crane 定时任务。

---

## 1. @Crane 注解全解析

### 1.1 注解元素

```java
// @Crane 只有一个参数：任务名
@Crane("incentiveStatusUpdate")
public void updateTaskStatus(String param) { ... }

// 完整语法（任务名是唯一参数）
// com.cip.crane.client.spring.annotation.Crane
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Crane {
    String value();   // 任务名，在控制台展示和触发时使用
}
```

### 1.2 方法签名规范

```java
// ✅ 标准签名（推荐）
@Crane("myJob")
public void myJob(String param) { ... }
// param：控制台下发的参数，可以是日期、批次号、JSON等

// ✅ 无参数签名（控制台不传参时）
@Crane("myJob")
public void myJob() { ... }

// ❌ 不推荐：返回值
@Crane("myJob")
public String myJob(String param) { return "done"; }
// Crane 框架不使用返回值，但功能上也能跑
```

### 1.3 任务所在类必须是 Spring Bean

```java
// ✅ 正确：@Service 使类成为 Spring Bean
@Service
@Slf4j
public class IncentiveTaskJob {
    @Crane("incentiveStatusUpdate")
    public void updateTaskStatus(String param) { ... }
}

// ❌ 错误：普通类，不是 Spring Bean
public class IncentiveTaskJob {
    @Crane("incentiveStatusUpdate")  // Crane SDK 找不到此 Bean
    public void updateTaskStatus(String param) { ... }
}
```

## 2. 标准任务模板

### 2.1 基本模板

```java
@Service
@Slf4j
public class MyBusinessJob {

    @Autowired
    private MyMapperService mapperService;

    @Crane("myBusinessJob")              // ① Crane 任务名
    public void execute(String param) {
        long startTime = System.currentTimeMillis();
        log.info("[myBusinessJob] 开始执行, param:{}", param);  // ② 开始日志

        try {
            doExecute(param);             // ③ 实际业务逻辑
        } catch (Exception e) {
            log.error("[myBusinessJob] 执行异常, param:{}", param, e);  // ④ 异常日志
            // 注意：不要 throw！让 Crane 认为任务执行完了，避免无限重试
        } finally {
            log.info("[myBusinessJob] 执行完毕, param:{}, 耗时:{}ms",
                param, System.currentTimeMillis() - startTime);  // ⑤ 结束日志
        }
    }

    private void doExecute(String param) {
        // 实际业务逻辑放这里
    }
}
```

### 2.2 带执行统计的模板

```java
@Crane("taskRewardPenaltyJob")
public void taskRewardPenaltyJob(String dateStr) {
    long startTime = System.currentTimeMillis();
    int successCount = 0, failCount = 0;
    log.info("[taskRewardPenaltyJob] 开始执行, dateStr:{}", dateStr);

    try {
        List<IncentiveTask> tasks = incentiveMapperService.queryPendingTasks(dateStr);
        log.info("[taskRewardPenaltyJob] 待处理任务数:{}", tasks.size());

        for (IncentiveTask task : tasks) {
            try {
                ccsTaskRewardService.processRewardPenalty(task);
                successCount++;
            } catch (Exception e) {
                failCount++;
                log.error("[taskRewardPenaltyJob] 处理任务失败, taskId:{}", task.getId(), e);
                // 单条失败不中断，继续处理下一条
            }
        }
    } catch (Exception e) {
        log.error("[taskRewardPenaltyJob] 任务执行异常, dateStr:{}", dateStr, e);
    } finally {
        log.info("[taskRewardPenaltyJob] 执行完毕, dateStr:{}, 成功:{}, 失败:{}, 耗时:{}ms",
            dateStr, successCount, failCount, System.currentTimeMillis() - startTime);
    }
}
```

## 3. 参数 param 的使用

### 3.1 参数的典型用途

| 参数值示例 | 用途 |
|-----------|------|
| `"2024-01-15"` | 指定处理哪天的数据（补跑用） |
| `"force=true"` | 强制重新处理（忽略已处理标记） |
| `""` 或 `null` | 不传参数，用当天日期 |
| `"pageSize=500"` | 覆盖默认批量大小 |

### 3.2 参数解析示例

```java
@Crane("ccsStaffLevelJob")
public void calcStaffLevel(String param) {
    // 解析参数：不传时用今天，传了就用指定日期
    String dateStr = StringUtils.isBlank(param) ?
        DateUtils.formatDate(new Date()) : param.trim();

    log.info("[ccsStaffLevelJob] 开始计算员工等级, dateStr:{}", dateStr);

    // 根据日期计算
    staffLevelService.calcStaffLevel(dateStr);
}
```

### 3.3 JSON 参数（复杂场景）

```java
@Crane("customJob")
public void customJob(String param) {
    // param = {"date":"2024-01-15","batchSize":200,"dryRun":true}
    JSONObject config = JSON.parseObject(StringUtils.isBlank(param) ? "{}" : param);

    String date = config.getString("date");
    int batchSize = config.getIntValue("batchSize") > 0 ?
        config.getIntValue("batchSize") : 200;       // 默认值
    boolean dryRun = config.getBooleanValue("dryRun");  // false if not set

    log.info("customJob param: date={}, batchSize={}, dryRun={}", date, batchSize, dryRun);
}
```

## 4. 一个类中多个任务

### 4.1 云办公 Job 示例

```java
// CloudOfficeStaffDesktopJob.java
@Service
@Slf4j
public class CloudOfficeStaffDesktopJob {

    @Crane("cloudOfficeStaffDesktopJob")        // 主任务：分配云桌面
    public void cloudOfficeStaffDesktopJob(String param) {
        log.info("[cloudOfficeStaffDesktopJob] 开始");
        cloudOfficeStaffDesktopService.distribute();
    }

    @Crane("cloudOfficeStaffDesktopCreateJob")  // 子任务：创建账号
    public void createDesktopAccount(String param) {
        log.info("[cloudOfficeStaffDesktopCreateJob] 开始");
        cloudOfficeStaffDesktopService.createAccounts();
    }

    @Crane("cloudOfficeStaffDesktopInitJob")    // 子任务：初始化
    public void initDesktop(String param) {
        log.info("[cloudOfficeStaffDesktopInitJob] 开始");
        cloudOfficeStaffDesktopService.initialize();
    }

    @Crane("cloudOfficeStaffDesktopRemindJob")  // 子任务：到期提醒
    public void remindDesktopExpiry(String param) {
        log.info("[cloudOfficeStaffDesktopRemindJob] 开始");
        cloudOfficeStaffDesktopService.remindExpiry();
    }
}
```

**设计建议**：把同一业务域的 Job 放在一个类中，但每个方法对应一个独立的 Crane 任务，可以分别配置执行时间。

### 4.2 激励任务 Job 示例


## 5. 分批处理大数据量

### 5.1 为什么要分批

```
员工等级计算：总员工数 10000
如果一次性加载：
  List<Staff> allStaff = staffMapper.queryAll();  // 10000 条 → 可能 OOM
  // 或者：数据库查询慢，超时

正确做法：分页处理
```

### 5.2 分页模板

```java
@Crane("ccsStaffLevelJob")
public void calcStaffLevel(String param) {
    int pageSize = lionService.defaultOrgStaffAmount != null ?
        lionService.defaultOrgStaffAmount : 200;     // 从 Lion 读取批量大小
    int pageNum = 1;
    int totalProcessed = 0;

    log.info("[ccsStaffLevelJob] 开始，pageSize:{}", pageSize);

    while (true) {
        // 分页查询（pageNum 和 pageSize）
        List<Staff> batch = staffMapper.queryByPage(pageNum, pageSize);
        if (CollectionUtils.isEmpty(batch)) {
            break;    // 没有更多数据，退出循环
        }

        // 处理这一批
        for (Staff staff : batch) {
            try {
                staffLevelService.calcLevel(staff);
                totalProcessed++;
            } catch (Exception e) {
                log.error("[ccsStaffLevelJob] 单条处理失败, staffId:{}", staff.getId(), e);
            }
        }

        log.info("[ccsStaffLevelJob] 第{}页处理完成，本批:{}，累计:{}",
            pageNum, batch.size(), totalProcessed);

        if (batch.size() < pageSize) {
            break;    // 最后一页
        }
        pageNum++;
    }

    log.info("[ccsStaffLevelJob] 全部完成，总处理:{}", totalProcessed);
}
```

## 6. 单条失败不中断整体

### 6.1 错误设计（单条异常终止整个 Job）

```java
// ❌ 危险：第5条失败就整个 Job 停了
@Crane("myJob")
public void myJob(String param) {
    List<Task> tasks = mapper.queryAll();
    for (Task task : tasks) {
        processTask(task);   // 第5条抛异常 → 后面的都不执行了
    }
}
```

### 6.2 正确设计（单条失败继续处理其他）

```java
// ✅ 正确：单条失败记录日志，不影响其他条
@Crane("myJob")
public void myJob(String param) {
    List<Task> tasks = mapper.queryAll();
    int success = 0, fail = 0;

    for (Task task : tasks) {
        try {
            processTask(task);
            success++;
        } catch (Exception e) {
            fail++;
            log.error("[myJob] 处理任务失败, taskId:{}", task.getId(), e);
            // 继续下一条，不 break
        }
    }

    log.info("[myJob] 完成，成功:{}, 失败:{}", success, fail);
    if (fail > 0) {
        // 有失败时可以发告警（可选）
        alertService.sendAlert("myJob 有 " + fail + " 条失败");
    }
}
```
